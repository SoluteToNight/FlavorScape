# 寻味地理页面功能说明（当前实现）

> 适用范围：当前代码库实际实现状态  
> 基准来源：`src/views/*`、`src/components/Navbar.vue`、`src/stores/app.js`、`backend/routers/*`  
> 用途：帮助后续功能完善、页面重构与联调开发  
> 说明：本文档以当前代码行为为准，不以设计稿中的理想状态替代实际实现。

## 1. 页面总览

项目当前是一个 Vue 3 单页应用，页面由 [src/App.vue](</E:/大学/大三下/GIS开发/Food/src/App.vue>) 承载，顶部固定导航为 [src/components/Navbar.vue](</E:/大学/大三下/GIS开发/Food/src/components/Navbar.vue>)，主体通过路由切换五个页面：

| 路由 | 页面 | 组件文件 | 核心职责 |
|:---|:---|:---|:---|
| `/` | 首页 | `src/views/HomeView.vue` | 建立第一印象，负责把用户导入地图主体验 |
| `/map` | 探索地图 | `src/views/MapView.vue` | 展示地理底图、生态区、传播路径、风味节点，是全站核心功能页 |
| `/library` | 风味基因库 | `src/views/LibraryView.vue` | 以卡片视图浏览全部风味节点，适合结构化对比 |
| `/narrative` | 时空叙事馆 | `src/views/NarrativeView.vue` | 按章节展示历史传播故事和路线缩略演示 |
| `/about` | 关于·方法论 | `src/views/AboutView.vue` | 说明数据来源和方法论，承担“信任说明页”角色 |

## 2. 跨页面公共能力

### 2.1 顶部导航栏

- 导航栏组件：`src/components/Navbar.vue`
- 作用：
  - 提供全站主导航
  - 提供全局搜索入口
  - 在首页和地图页切换不同透明度视觉状态
- 当前导航项：
  - 首页
  - 探索地图
  - 风味基因库
  - 时空叙事馆
  - 关于方法论

### 2.2 全局搜索

- 入口位置：导航栏右侧搜索按钮
- 接口：`GET /api/search?q=`
- 当前可选结果类型：
  - `node`：风味节点
  - `route`：传播路径
- 当前行为：
  - 选择节点后调用 `appStore.selectNode()`
  - 选择路线后调用 `appStore.selectRoute()`
  - 然后统一跳转到 `/map`
- 现状限制：
  - 不能直接搜索生态区
  - 不能直接搜索叙事章节
  - 搜索结果的主要落点只有地图页

### 2.3 全局状态管理

状态集中在 [src/stores/app.js](</E:/大学/大三下/GIS开发/Food/src/stores/app.js>)：

- `flavors`：全部风味节点数据
- `selectedNode`：当前选中的风味节点
- `selectedRoute`：当前选中的传播路径
- `selectedEcozone`：当前选中的生态区
- `currentChapter`：叙事馆当前章节索引
- `layerVisibility`：上下文触发后的实际图层可见性
- `layerEnabled`：用户主动开关状态
- `l1Emphasis`：是否强调生态区边界

当前交互规则：

- 节点、路线、生态区三者互斥
- 选中节点或路线时，L2 传播路径层会被激活
- 选中生态区时，L2 会隐藏，L1 生态区边界会增强显示

## 3. 页面说明

### 3.1 首页 `/`

文件： [src/views/HomeView.vue](</E:/大学/大三下/GIS开发/Food/src/views/HomeView.vue>)

#### 页面定位

首页是“入口页”和“气氛页”，主要职责不是承载复杂功能，而是：

- 让用户快速理解主题是“风味与地理的关系”
- 用视觉动效建立博物志式气质
- 把用户导向地图页

#### 当前实现内容

- 全屏 `canvas` 粒子背景
- 中央主文案和按钮“开启地图”
- 底部 ticker 文本轮播
- 鼠标移动对粒子轨迹产生轻微扰动

#### 数据来源

- 无后端接口依赖
- ticker 文案和路径定义写死在组件内部

#### 用户操作路径

1. 进入首页
2. 观看背景动效和文案
3. 点击“开启地图”
4. 跳转到 `/map`

#### 对后续开发的意义

- 首页承担“首次进入转化”职责，适合继续强化品牌表达和入口效率
- 这里适合承接“推荐入口”“精选故事”“热门节点”之类的快速跳转能力

#### 当前开发关注点

- 页面数据全部是静态写死，缺少可配置能力
- 首页没有直接挂接推荐节点、推荐路线或推荐章节
- ticker 内容没有接入真实业务数据
- 目前核心 CTA 只有“进入地图”，缺少“进入叙事”“进入基因库”等分流入口

### 3.2 探索地图 `/map`

文件： [src/views/MapView.vue](</E:/大学/大三下/GIS开发/Food/src/views/MapView.vue>)  
配套检视器： [src/components/MapInspector.vue](</E:/大学/大三下/GIS开发/Food/src/components/MapInspector.vue>)

#### 页面定位

地图页是全站核心页面，负责把“自然环境、传播路径、风味节点”放到同一空间里，完成探索、检视和联动展示。

#### 当前实现内容

- 使用 MapLibre 渲染自然地形底图
- 使用 Deck.gl 渲染传播路径、流动粒子和风味节点
- 左侧显示图层图例和图层开关
- 右侧显示 MapInspector 检视面板
- 支持点击节点、路径、生态区查看详情
- 支持工具提示、缩放、漫游、飞行定位

#### 图层结构

- `L0`：自然地形底图、海岸线、河流
- `L1`：生态区边界
- `L2`：传播路径与粒子流光
- `L3`：风味节点

#### 数据来源

页面初始化时调用：

- `GET /api/flavors`
- `GET /api/routes`
- `GET /tiles/status`

地图资源运行时继续依赖：

- `GET /tiles/raster/{z}/{x}/{y}.png`
- `GET /tiles/vector/coastline`
- `GET /tiles/vector/rivers`
- `GET /tiles/vector/ecoregions`

搜索跳转还依赖：

- `GET /api/search?q=`

#### 页面内主要交互

- 点击风味节点：
  - 选中节点
  - 右侧显示雷达图、维度条形图、食材基因、同族节点
  - 地图 `flyTo` 到节点附近
- 点击传播路径：
  - 选中路线
  - 右侧显示路线说明和路径坐标
  - 地图 `fitBounds` 到路径范围
- 点击生态区边界：
  - 选中生态区
  - 右侧显示生态档案和模拟气候图
  - 生态区边界增强，传播路径层关闭
- 点击左侧图层开关：
  - 控制 L0-L3 是否参与显示

#### 检视器承担的功能

MapInspector 当前是地图页最重要的信息面板，承担三种对象的详情展示：

- 风味节点详情
- 生态区详情
- 传播路径详情

这意味着地图页不仅是“看地图”，也是全站最主要的详情页。

#### 对后续开发的意义

- 地图页是最适合承接新功能的页面
- 几乎所有“搜索”“专题跳转”“详情查看”“图层联动”最终都应落到这里
- 如果后续要做更强的 GIS 分析、故事联动、推荐探索、路径筛选，地图页会是主战场

#### 当前开发关注点

- `MapView.vue` 体积较大，地图初始化、图层构建、交互同步、动画循环都集中在一个文件，后续维护成本会继续上升
- 生态区气候图是模拟数据，不是真实后端气候数据
- 路线说明文案是组件内通用模板，不是按路线生成的结构化内容
- `/tiles/status` 只在进入页面时请求一次，缺少更明确的加载状态与失败恢复逻辑
- 图层开关和上下文触发逻辑已经存在，但状态语义较多，后续扩展时要避免交互混乱
- 地图页已经兼具“探索页”和“详情页”双重职责，继续加功能前需要先判断是否要拆分侧边信息架构

### 3.3 风味基因库 `/library`

文件： [src/views/LibraryView.vue](</E:/大学/大三下/GIS开发/Food/src/views/LibraryView.vue>)

#### 页面定位

基因库页是全站的“结构化浏览页”。它不强调空间漫游，而强调把所有风味节点以统一卡片形式展开，方便用户批量浏览和横向比较。

#### 当前实现内容

- 顶部分类筛选按钮
- 节点总数统计
- 风味卡片网格
- 卡片内包含：
  - 城市
  - 地域与生态信息
  - 两个主风味维度数值
  - 雷达图
  - 生态区标签
- 点击卡片跳转地图页并选中对应节点

#### 数据来源

- `GET /api/flavors`

#### 用户操作路径

1. 进入基因库页
2. 按分类筛选风味节点
3. 浏览卡片并进行心智对比
4. 点击某张卡片
5. 跳转到地图页并打开该节点详情

#### 对后续开发的意义

- 基因库页适合扩展“对比”“排序”“收藏”“标签过滤”类功能
- 它可以成为用户做批量认知的入口，再把深度分析导入地图页
- 如果后续要做面向研究或教学的浏览体验，这一页的价值会很高

#### 当前开发关注点

- 分类项是前端硬编码，未从数据模型动态派生
- 当前只有单条件筛选，没有搜索、排序、组合过滤、对比模式
- 点击卡片后仍然依赖地图页承接详情，当前页面自身不提供深度信息
- 卡片循环的 `key` 目前使用 `d.city`，存在城市重复时的稳定性风险

### 3.4 时空叙事馆 `/narrative`

文件： [src/views/NarrativeView.vue](</E:/大学/大三下/GIS开发/Food/src/views/NarrativeView.vue>)

#### 页面定位

叙事馆是“线性阅读页”。与地图页的自由探索不同，它负责把若干典型传播事件组织成章节，让用户按故事线理解风味传播和环境影响。

#### 当前实现内容

- 左侧章节导航
- 右侧章节正文
- 顶部缩略路线动画画布
- 章节引用、来源和进度点
- 最后一章显示“打开全地图自由探索”按钮

#### 数据来源

- `GET /api/chapters`
- `GET /api/routes`

#### 当前交互逻辑

- 用户点击左侧章节点切换当前章节
- 章节切换后：
  - 更新标题、时间、正文、引用、来源
  - 重新绘制与该章节绑定的传播路线缩略图
- 到达最后一章时，可跳转到地图页继续探索

#### 与地图页的关系

- 叙事馆是“先讲故事，再导向探索”
- 地图页是“先自由探索，再看对象细节”
- 当前两者之间的关系已经建立，但联动仍然偏弱

#### 对后续开发的意义

- 这一页最适合承接专题策展、时间线、章节化讲解、课程演示
- 如果后续要做“某一食材如何扩散”“某一菜系如何分化”的专题，这里会是主要载体

#### 当前开发关注点

- 当前章节切换主要依赖点击，不支持滚动式章节驱动或自动播放
- 章节状态保存在全局 store，中途跳页再回来会保留上次章节，需要确认这是否符合产品预期
- 章节跳转到地图页时，还没有把对应路线或节点自动带入地图上下文
- 缩略图是前端 canvas 自绘，表现力足够轻，但后续如果要支持更复杂地图叙事，可能需要升级为真正地图组件

### 3.5 关于·方法论 `/about`

文件： [src/views/AboutView.vue](</E:/大学/大三下/GIS开发/Food/src/views/AboutView.vue>)

#### 页面定位

关于页是全站的“信任页”和“说明页”，主要回答两个问题：

- 数据从哪里来
- 这个项目如何理解和组织“风味地理”

#### 当前实现内容

- 标题与导语
- 数据来源列表
- 每条来源可打开外部链接
- 一个可展开的“LLM 辅助清洗说明”

#### 数据来源

- `GET /api/data-sources`

#### 页面作用

- 为用户提供方法论背书
- 为开发者和评审者提供数据出处入口
- 在整体叙事中补足“这不是纯视觉项目，而是有数据来源支撑”的说明

#### 对后续开发的意义

- 适合扩展成更完整的“数据字典/来源字典/计算说明”页面
- 适合加入数据库表说明、字段解释、更新周期、处理流程图
- 也适合成为未来的开发文档入口页

#### 当前开发关注点

- 当前页面展示层较轻，更多是说明性内容，缺少和实际数据库字段、接口返回结构的一一对应
- “LLM 辅助清洗说明”当前更像概念陈述，不完全等于现有代码中的真实处理流程
- 这一页现在更接近产品说明页，还不是严格的技术方法文档页

## 4. 页面联动关系

当前页面之间的主路径如下：

1. 首页负责把用户送入地图页
2. 地图页承接搜索、选择、详情展示
3. 基因库页负责批量浏览节点，再把用户送入地图页查看细节
4. 叙事馆负责章节化讲故事，再把用户送入地图页自由探索
5. 关于页负责补足信任和说明，不直接承载深度交互

可以把当前结构理解为：

- 首页：入口
- 地图页：核心主场
- 基因库：结构化浏览入口
- 叙事馆：专题叙事入口
- 关于页：说明与背书

## 5. 适合下一步开发的方向

如果后续要继续完善功能，建议优先按页面职责推进，而不是平均用力：

### 第一优先级：地图页

- 增强详情内容的真实性和结构化程度
- 补足加载、异常、空状态
- 拆分大组件，降低维护成本
- 明确“探索”和“详情”边界

### 第二优先级：基因库页与叙事馆

- 基因库页增强筛选、排序、对比
- 叙事馆增强章节到地图的联动和专题表达

### 第三优先级：首页与关于页

- 首页增强导流能力和动态推荐
- 关于页补足真实方法链路、数据字典和开发说明

## 6. 建议的后续文档拆分

如果接下来要持续开发，建议在本文基础上继续拆出三类文档：
