# 寻味地理总体设计文档配图规划（重梳理版）

本文档用于重新梳理《寻味地理》总体设计文档中应补充的图。它不是最终插图本身，而是“画图清单”和“插图位置说明”。后续真正绘图时，应先按本清单确定图号、图名、插入位置和表达重点，再进入 SVG/PNG 绘制。

## 一、总体判断

前一版只画了少量概念图和结构图，数量和类型都不够。参考设计书这类总体设计文档通常不会只放“系统架构图”和“E-R 图”，还会在需求、总体设计、数据库、功能、接口、界面等章节中持续放置结构图、流程图、模块图、数据库图、调用图、运行环境图和界面图。

结合本项目实际，图的重点不应照抄参考书里陈旧的框图，而应围绕以下几类内容展开：

1. **项目定位类图**：说明本系统不是餐馆点评或交易系统，而是风味知识 WebGIS 与数据资产原型。
2. **数据对象类图**：说明生态区、风味节点、食材、传播事件、叙事章节、数据来源之间的关系。
3. **系统架构类图**：说明 Vue、MapLibre、Deck.gl、FastAPI、PostGIS、瓦片服务、专题 JSON 如何协同。
4. **数据流程类图**：说明基础地理数据、业务数据、食材传播数据、LLM 辅助生产数据如何整理、审核、入库和发布。
5. **数据库类图**：说明空间数据库、业务实体、关联表、索引和建库维护流程。
6. **功能模块类图**：说明各页面模块和通用能力，不只是列页面名称。
7. **接口调用类图**：说明浏览器页面、API、查询层、瓦片服务、数据库和数据文件之间的调用关系。
8. **界面类图**：参考设计书往往会放大量界面图；本项目应使用真实页面截图或高保真线框图，并做标注。

建议最低成稿数量为 **28 张必画图**。如果篇幅允许，可增加 **8 张增强图**。不建议把每个小标题都硬塞一张图，否则设计书会显得碎。

## 二、图件分级

- **A 级，必画**：缺少后会影响设计书完整性，建议全部补上。
- **B 级，建议画**：能够显著增强专业性和产业化表达，但可根据篇幅合并。
- **C 级，可替换**：可以用真实系统截图替代，也可以用简单示意图表达。

## 六、绘图风格建议

1. **结构图少用斜线**：尽量使用分层、分列、正交连接线。
2. **数据库图要覆盖真实表**：不能只画概念名，至少要覆盖 `schema.sql` 中的主要表和关键外键。
3. **界面章节优先用真实截图**：参考设计书里的界面图虽然旧，但它的意思是“让老师看到系统真的长什么样”。本项目界面图应使用真实运行截图再加标注。
4. **食材传播必须有路径截图**：如果路线可视化还没修好，界面图会暴露问题；应先修可视化，再截图。
5. **产业化表达要放在数据闭环、来源治理、专题模板、接口复用中**：不要每张图都写产业化，容易变成口号。
6. **每张图标题要像设计书图题**：例如“图 3-2 系统总体架构图”，不要用花哨标题。
7. **PNG 用于 Word 插入，SVG 用于后期编辑**：最终交付时每张图都应同时保存 PNG 和 SVG。
