## 7. `/brand` 产品生成器

文件：`src/views/BrandStoryView.vue`

产品定位：

把 GIS 和食材产地信息转成可展示、可导出、可传播的视觉产品。

当前功能：

- 使用 `html2canvas` 导出当前产品画布为 PNG。
- 当前捕获区域统一为 `#capture-zone`。
- 使用 MapLibre 加载真实栅格底图：`/tiles/raster/{z}/{x}/{y}.png`。
- 使用 `/ingredients/pepper-realistic.png` 作为汉源花椒示例图。
- 有多个产品形态，通过 `currentView` 切换。

当前视图：

- `atlas`：地理志，完整 StoryMap 联动布局。
- `brand`：品牌公示，3:4 纵向海报卡片。
- `chain`：产业链，纵向流程图。
- `archive`：风物档案，产地环境、历史传播、加工技艺、认证检测、代表菜品综合档案。

当前示例产品：

- 名称：汉源花椒
- 图片：`/ingredients/pepper-realistic.png`
- 核心产区：四川雅安汉源 · 大渡河干热河谷
- 节点：产地、加工、仓储、市场、餐桌等
- 输出场景：品牌官网、展厅、包装内页、电商详情页、门店公示牌、展会展示板

注意：

- `/brand` 目前是一个比较大的单文件组件。
- 已经从纯 SVG 示意图推进到真实 MapLibre 底图叠加节点。
- 继续开发时优先保持可导出能力，不要把 UI 改成只能屏幕展示。

## 8. `/spread` 食材传播图谱

文件：`src/views/IngredientSpreadView.vue`

产品定位：

展示多食材历史传播路径，既能总览，也能进入单个食材时间线。

当前功能：

- 默认进入“全部”总览。
- 食材 tab：全部、辣椒、稻米、八角等。
- 右侧为食材信息和时间线。
- 地图使用 MapLibre HYP 栅格底图。
- 地图节点使用食材图片图标。
- 总览模式下多食材同时显示，线条低透明。
- 单食材模式下显示完整路径、首传节点、重复地区过滤提示。
- 对同一地区重复历史记录做 dedupe，地图只绘制每个地区首次传入节点。
- 支持点击侧栏事件定位地图节点。
- 路径会按陆路、水路、海路等语义做不同弧线风格。

重要数据行为：

- `sortedTimeline()` 会过滤没有坐标的事件并按年份排序。
- `dedupeFirstArrival()` 使用 `region_key / region / province / location` 归一化地区。
- `stableEventKey()` 用事件 ID 或年份/地区/位置/路线生成稳定 key。
- `ingredientImage()` 优先使用 JSON 中的 `image_url`。

## 9. 食材传播 API

文件：`backend/routers/ingredient_spread.py`

路由挂载：

`backend/main.py` 中：

```python
from .routers.ingredient_spread import router as ingredient_router
app.include_router(ingredient_router)
```

接口：

- `GET /api/ingredients/spread`
- `GET /api/ingredients/spread/{ingredient_id}`
- `GET /api/ingredients/spread/{ingredient_id}/path`

数据来源：

```text
data/ingredient/*.json
```

当前食材：

- `pepper.json`：辣椒
- `rice.json`：稻米
- `star_anise.json`：八角

JSON schema 要点：

```json
{
  "ingredient_id": "pepper",
  "name": "辣椒",
  "name_en": "Chili Pepper",
  "species": "Capsicum annuum",
  "origin": "美洲",
  "origin_coordinates": [-99.13, 19.43],
  "summary": "...",
  "color": "#E5394E",
  "image_url": "/ingredients/pepper-realistic.png",
  "timeline": [
    {
      "year": 1591,
      "dynasty": "明代后期",
      "location": "浙江",
      "region_key": "浙江",
      "coordinates": [120.15, 30.28],
      "historical_name": ["番椒"],
      "source_literature": "《遵生八笺》",
      "event_type": "首次记载",
      "route": "华东沿海传入",
      "notes": "..."
    }
  ]
}
```

新加食材时至少要补：

- `ingredient_id`
- `name`
- `origin`
- `origin_coordinates`
- `summary`
- `color`
- `image_url`
- `timeline[].year`
- `timeline[].location`
- `timeline[].coordinates`
- `timeline[].event_type`
- `timeline[].route`
- `timeline[].notes`

建议每条 timeline 都加 `region_key`，否则去重会退化到 `location`。

## 15. 产品方向说明

项目原始方向是“饮食文化 WebGIS 展示系统”，现在演进为：

```text
地方食材品牌故事地图
+ 轻量溯源可视化产品生成器
+ 食材历史传播图谱
```

这不是普通后台系统，也不是单纯地图 demo。后续设计应保持：

- 地理手稿感
- 博物志/档案感
- 可导出、可传播、可交付
- 地图和叙事结合
- 食材品牌、文旅展示、电商详情页、展会展板可复用

不要把界面改成通用 SaaS 后台风格。
