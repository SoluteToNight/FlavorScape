# 寻味地理开发进度同步：产品化原型重构

## 一、当前开发方向

目前项目已经从早期的 WebGIS 展示型系统，逐步转向“地方食材品牌故事地图 + 轻量溯源可视化产品生成器”的方向。

也就是说，项目不再只是把地图、节点和路径放在网页上，而是希望把 GIS 能力转化为可以被品牌方、地方特产企业、文旅展示场景、电商详情页和展会展板直接使用的可视化产品。

当前保留了项目原有的米色网格纸、地理手稿、食材档案感视觉底色，但正在减少“后台系统感”，强化“可交付、可导出、可传播”的产品属性。

## 二、产品生成器页面重构

本轮重点重构的是 `/brand` 页面，也就是现在的“生成器”页面。

过去这个页面更像一个普通网页展示页，当前已经改为支持两种明确的产品形态切换：

1. 地理志
2. 品牌公示

这两种形态不是简单换样式，而是通过 `currentView` 状态控制不同的 DOM 结构。

当用户选择“地理志”时，页面恢复为全屏 StoryMap 联动布局；当用户选择“品牌公示”时，页面中央会切换为 3:4 的纵向海报卡片。

## 三、地理志视图

“地理志”视图定位为完整的食材空间叙事页面，适合用于品牌官网、展厅大屏、项目演示和食材文化传播。

当前布局由三部分组成：

- 左侧是食材身份卡片，展示食材名称、简介、产地、价值和输出场景。
- 中央是地图叙事主画布，展示产地、加工、集散、餐桌、外部市场等节点。
- 右侧是纵向叙事时间线，用点状虚线和呼吸发光节点表达故事推进。

中央地图卡片已经做了层级叠放处理，不再是机械的三栏格栅。地图卡片通过负 margin 与左右两侧卡片产生轻微重叠，使页面更像一张经过设计的地理叙事图板，而不是后台面板。

地图中的节点目前使用食材图标作为视觉元素，路线使用带有渐变和粒子动效的曲线路径，用来表达从产地到消费市场的流动过程。

## 四、品牌公示视图

“品牌公示”视图定位为更轻量、更适合传播的产品输出形态。

当前页面会切换为 3:4 纵向海报卡片，整体接近小红书海报、电商详情图、门店公示牌和展会展示板的使用场景。

当前海报内容包括：

- 食材名称与公示标题
- 真实风格食材图
- 轻量溯源路线图
- 产地、工艺、应用、附件四类信息卡片
- 二维码占位区
- 完整地理志入口提示

这个视图的意义在于，它把 GIS 和溯源信息压缩成一张可以直接传播的视觉产品，而不是要求用户进入复杂系统后再理解数据。

## 五、PNG 导出能力

已经引入 `html2canvas`，并为页面中的“下载 PNG”按钮接入导出逻辑。

当前实现方式如下：

- 当前可见的产品画布统一使用 `#capture-zone` 标记。
- 点击“下载 PNG”后，程序会抓取当前画布区域。
- 导出时使用 `scale: 2`，生成高分辨率 PNG 图片。
- 导出的内容会随当前视图变化：如果当前是“地理志”，导出 StoryMap 画布；如果当前是“品牌公示”，导出 3:4 海报卡片。

这一步非常关键，因为它让项目具备了真正的产品输出能力。后续可以继续扩展为模板化生成器，例如批量生成产地公示图、品牌故事海报、溯源报告封面、电商详情页长图等。

## 六、视觉布局调整

本轮也对页面视觉结构做了调整，目标是减少“生硬后台感”。

主要变化包括：

- 保留米色网格纸风格，继续强化地理手稿和食材档案感。
- 中央地图卡片加入负 margin，与左右卡片形成层级叠放。
- 右侧普通列表改成纵向叙事时间线。
- 时间线使用点状虚线和呼吸发光节点，表达路径推进和故事流动。
- 地图节点改为食材图标，不再使用普通数字节点。
- 路线采用渐变曲线和粒子流动效果，增强传播路径的方向感。

目前这一版不是最终视觉稿，但已经从“后台页面”向“产品化视觉模板”迈出了一步。

- 已用 Chrome headless 截图检查两个视图。

截图文件位于：

## 八、当前不足

虽然当前已经具备产品原型结构，但距离真正可交付的商业化视觉产品还有差距。

主要不足包括：

- 当前地图仍是抽象 SVG 示意图，不是真实 GIS 地理底图。
- 食材、产地、工艺、流通和餐桌场景的数据还不够完整。
- 海报已经有产品形态，但版式精度、图像质量和真实内容密度还需要继续提升。
- 目前只有单个示例食材，后续需要支持更多地方食材模板。
- 导出能力已经具备，但还没有形成完整的模板管理、样式选择和批量生成流程。

## 九、下一步建议

下一阶段可以重点做三件事。

第一，补充真实数据。

需要为每种食材整理产地、加工节点、流通节点、餐饮应用、图片素材、认证材料和说明文字。只有数据扎实，产品化模板才不会显得空。

第二，继续提升地图视觉。

目前可以先保留 SVG 产品化画布，但后续应逐步接入真实行政区划、产区范围、地形底图或自定义矢量底图，让“地理志”真正具备 GIS 价值。

第三，完善模板生成能力。

后续可以增加模板选择、食材选择、风格选择、内容编辑、批量导出等功能，让项目从“一个展示网站”继续推进为“地方食材可视化产品生成工具”。

## 十、阶段总结

当前项目已经完成一次重要转向：从单纯展示地图和食材信息，转向生成可传播、可导出、可复用的地方食材品牌视觉产品。

这次重构的核心价值不在于页面多了几个组件，而在于项目开始具备“输出产品”的能力。后续只要继续补充真实数据、提高地图精度和完善模板体系，就可以把“寻味地理”逐步建设成一个面向地方食材品牌、文旅展示和轻量溯源服务的产品化 GIS 应用。
