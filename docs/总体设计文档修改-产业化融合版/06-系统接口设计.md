# 系统接口设计

本章对“寻味地理”风味博物志 WebGIS 系统的外部接口、前后端业务接口、地图瓦片接口、系统状态接口、数据流和错误处理约定进行说明。接口设计以浏览器端 WebGIS 应用为服务对象，重点保证地图展示、风味浏览、传播路径展示、食材传播、叙事阅读和数据来源说明能够稳定获取数据。面向后续产业化演进，接口还应保持结构稳定和语义清楚，使同一批风味数据能够被专题页面、展陈大屏或外部系统复用。

## 外部接口

### 与外部数据源的接口

#### Natural Earth 数据接口

系统使用 Natural Earth 高程分层设色栅格底图和物理矢量数据作为基础地理数据来源。该接口属于离线数据接入接口，不在用户访问时实时调用外部服务。

| 接口对象 | 数据内容 | 接入方式 | 系统用途 |
| --- | --- | --- | --- |
| 高程分层设色栅格 | 全球自然地形背景 | 本地文件读取 | 生成栅格地图瓦片 |
| 海岸线 | 全球海岸线矢量 | 本地矢量数据加载 | 地图物理边界展示 |
| 河流 | 全球河流矢量 | 本地矢量数据加载 | 地图水系参照 |
| 陆地轮廓 | 全球陆地矢量 | 本地矢量数据加载 | 地图陆海背景参照 |

外部数据更新时，应先完成数据格式检查、坐标系检查和基本渲染检查，再进入系统数据目录。

#### WWF TEOW 数据接口

系统使用 WWF TEOW 全球陆地生态区划作为生态地理单元来源。该接口同样属于离线数据接入接口，生态区数据经整理后进入空间数据库。

| 接口对象 | 数据内容 | 接入方式 | 系统用途 |
| --- | --- | --- | --- |
| 生态区边界 | MultiPolygon 面状几何 | 空间数据库导入 | 生态区图层和点击查询 |
| 生态区属性 | 名称、编码、生物群系、生物地理界、面积 | 属性字段映射 | 详情面板和数据解释 |
| 中文补充属性 | 中文名称、中文类型说明 | 人工整理或映射 | 面向公众展示 |

生态区数据接入后，应保证边界几何有效、坐标系一致、属性字段可读。

#### 媒体与文本来源接口

系统涉及图片材料、饮食文化材料、地方美食志、古籍和研究资料等来源。此类数据不应由前端直接抓取，而应在系统数据整理阶段确定可用资源和引用说明。

| 数据类型 | 接口要求 |
| --- | --- |
| 图片资源 | 记录来源链接、授权说明和用途，无法稳定加载时提供占位展示 |
| 文本材料 | 记录材料名称、作者或来源、引用内容和整理用途 |
| LLM 辅助结果 | 记录原始材料、抽取结果、审核状态和入库状态 |

外部数据接入不应直接等同于线上实时调用。当前系统以离线整理、入库和本地服务输出为主，能够降低外部服务不可用对页面展示的影响。后续若接入第三方图片、知识库或内容审核服务，应在服务层进行缓存和降级处理，避免前端直接依赖不稳定外部资源。

### 与浏览器的接口

浏览器通过 HTTP 请求访问前端静态资源、业务数据接口和地图瓦片接口。系统应支持现代浏览器的 JavaScript、WebGL、JSON 解析和图片瓦片加载能力。

浏览器端接口交互应满足以下要求：

1. 所有业务数据接口返回 JSON 数据；
2. 栅格瓦片接口返回 PNG 图片；
3. 矢量图层接口返回 GeoJSON 或 MVT 数据；
4. 接口响应应设置合理缓存策略，减少重复请求；
5. 生产环境应通过 HTTPS 提供访问能力；
6. 面向外部复用的接口应预留版本号、权限和调用统计机制。

## 内部接口

### 前后端接口设计

#### 数据格式约定

系统前后端接口主要采用 JSON 格式交换业务数据。接口字段命名保持稳定，前端不直接依赖数据库内部字段名。

通用约定如下：

| 项目 | 约定 |
| --- | --- |
| 字符编码 | UTF-8 |
| 时间与年代 | 以展示型字符串为主，必要时补充排序字段 |
| 坐标顺序 | 经度在前，纬度在后 |
| 空间几何 | 点以坐标数组表达，路线以坐标数组或矢量格式表达 |
| 颜色 | 使用十六进制颜色字符串 |
| 空结果 | 返回空数组或空对象，不返回未定义结构 |

#### 业务数据接口

##### GET `/api/flavors`

接口用途：获取风味节点展示数据，用于地图节点展示、风味基因库卡片和搜索结果对象定位。该接口当前由数据库 `flavor_genotype`、`recipe_link` 和 `ingredient` 等表组合生成，但前端不需要感知内部表结构。

请求参数：无。

响应类型：JSON 数组。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 风味节点稳定标识 |
| `city` | string | 城市或地域名称 |
| `dish` | string | 菜品名称 |
| `dish_family` | string | 食物类型 |
| `region` | string | 区域标签 |
| `eco` | string | 生态展示标签 |
| `eco_name_ref` | string | 关联生态区名称 |
| `scores` | number[] | 六维风味向量展示数组 |
| `primary` | string[] | 主风味标签 |
| `vals` | number[] | 主风味数值 |
| `color` | string | 展示颜色 |
| `cat` | string | 风味分类 |
| `ingredients` | string[] | 食材名称列表 |
| `coordinates` | number[] | 节点坐标 |
| `genome_vector` | object | 六维风味向量对象 |
| `description` | string | 风味描述 |
| `bubbleImage` | string | 节点图片或占位图 |
| `bubbleImageSource` | string | 图片来源 |

##### GET `/api/routes`

接口用途：获取传播路径列表，用于地图路线展示、流光动画和叙事章节关联。

请求参数：无。

响应类型：JSON 数组。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `name` | string | 路线名称 |
| `color` | string | 路线展示颜色 |
| `type` | string | 路线类型，取值为 land 或 sea |
| `path` | number[][] | 路径坐标数组 |

##### GET `/api/search`

接口用途：根据关键词搜索风味节点和传播路径，用于全局搜索和地图跳转。

请求参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `q` | string | 否 | 搜索关键词 |

响应类型：JSON 数组。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `type` | string | 结果类型，node 或 route |
| `label` | string | 结果主标题 |
| `sub` | string | 结果副标题 |
| `color` | string | 结果颜色 |
| `data` | object | 可直接用于地图选择的对象数据 |

当关键词为空时，接口返回空数组。

##### GET `/api/chapters`

接口用途：获取时空叙事馆章节列表。

请求参数：无。

响应类型：JSON 数组。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | number | 章节编号 |
| `title` | string | 章节标题 |
| `date` | string | 年代标签 |
| `body` | string | 正文内容 |
| `cite` | string | 引用说明 |
| `source` | string | 来源 |
| `routeName` | string | 关联传播路线 |

##### GET `/api/data-sources`

接口用途：获取系统数据来源列表，用于关于与方法说明页面展示。

请求参数：无。

响应类型：JSON 数组。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `name` | string | 数据来源名称 |
| `desc` | string | 来源说明 |
| `color` | string | 展示颜色 |
| `url` | string | 来源链接 |

##### GET `/api/ingredients/spread`

接口用途：获取可用于食材传播页面的食材主题概要列表，包括食材编号、名称、物种、起源地、起源坐标、摘要、主题色、事件数量和年份范围。

请求参数：无。

响应类型：JSON 数组。前端使用该接口生成食材选择列表，并决定默认展示的传播主题。

##### GET `/api/ingredients/spread/{ingredient_id}`

接口用途：获取单个食材的完整传播数据，包括起源信息、时间线事件、参考文献和可视化参数。该接口主要服务于食材传播页面的右侧说明、时间线和地图节点渲染。

路径参数：`ingredient_id`，表示食材主题编号。

异常情况：当食材编号不存在时返回 404，并给出可用编号提示。

##### GET `/api/ingredients/spread/{ingredient_id}/path`

接口用途：获取单个食材按时间顺序生成的路径坐标和路径段信息。前端可使用该接口绘制传播路径、节点标记和阶段性镜头移动。

路径参数：`ingredient_id`，表示食材主题编号。

设计要求：接口返回的路径用于宏观传播叙事，不表示精确历史轨迹；每个节点应尽量关联时间、地点、路线描述和来源信息，便于后续进行专题审核和内容复用。

#### 瓦片服务接口

##### GET `/tiles/raster/{z}/{x}/{y}.png`

接口用途：按 XYZ 瓦片规则获取 Natural Earth 栅格底图瓦片。

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `z` | number | 缩放等级 |
| `x` | number | 瓦片横向编号 |
| `y` | number | 瓦片纵向编号 |

响应类型：PNG 图片。

异常情况：

| 状态 | 说明 |
| --- | --- |
| 400 | 缩放等级超出服务范围 |
| 503 | 栅格底图尚不可用 |

对于超出栅格边界的请求，系统应返回透明占位瓦片，避免地图渲染中断。

##### GET `/tiles/vector/{layer}`

接口用途：获取指定矢量图层的 GeoJSON 数据。

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `layer` | string | 图层名称 |

可用图层包括 coastline、rivers、land、ecoregions 等。

响应类型：GeoJSON FeatureCollection。

异常情况：

| 状态 | 说明 |
| --- | --- |
| 404 | 请求图层不存在或尚未加载 |

##### GET `/tiles/vector`

接口用途：获取当前可用矢量图层列表及要素数量。

请求参数：无。

响应示例结构：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `layers` | object | 图层名称与要素数量映射 |

##### GET `/tiles/mvt/{z}/{x}/{y}.pbf`

接口用途：获取生态区边界矢量瓦片，用于后续高性能地图图层展示。

路径参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `z` | number | 缩放等级 |
| `x` | number | 瓦片横向编号 |
| `y` | number | 瓦片纵向编号 |

响应类型：Mapbox Vector Tile 二进制数据。

当瓦片内无生态区要素或查询异常时，接口应返回空矢量瓦片响应，避免前端地图报错。

##### GET `/tiles/status`

接口用途：获取瓦片服务状态，用于地图初始化检查和调试显示。

请求参数：无。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `raster_ready` | boolean | 栅格底图是否可用 |
| `vector_layers` | string[] | 已加载矢量图层 |
| `tile_cache` | object | 瓦片缓存状态 |

#### 系统状态接口

##### GET `/health`

接口用途：检查后端服务是否可用。

请求参数：无。

主要响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | 服务状态 |
| `raster_ready` | boolean | 栅格底图是否可用 |
| `vector_layers` | string[] | 已加载矢量图层 |

### 子系统间的数据流设计

#### 数据查询流

用户访问页面后，浏览器根据页面类型请求业务数据或瓦片数据。地图页加载风味节点、传播路径、底图瓦片和矢量图层；风味基因库加载风味节点；时空叙事馆加载章节和路线；食材传播页面加载食材主题、完整时间线和路径坐标；关于页面加载数据来源。后端从数据库、专题 JSON 或基础数据文件读取数据，经格式化后返回浏览器。

#### 搜索定位流

用户输入关键词后，浏览器调用搜索接口。后端返回节点或路线类型的搜索结果。用户点击结果后，前端将结果对象写入当前选择状态，并跳转或定位到地图对应对象。

#### 数据写入流

当前用户界面不提供在线编辑入口。风味节点、传播事件、食材、食材传播专题和叙事章节通过离线整理、审核和批量导入流程进入数据库或专题数据文件。后续如建设管理后台，应另行设计登录、权限、审核、版本发布和写入接口。

### 接口错误码约定

| 状态码 | 适用场景 | 处理方式 |
| --- | --- | --- |
| 200 | 请求成功 | 返回目标数据 |
| 400 | 请求参数不符合要求 | 返回错误说明，前端提示用户 |
| 404 | 请求资源不存在 | 返回错误说明，前端显示空状态或提示 |
| 503 | 依赖数据或服务尚不可用 | 前端显示加载失败或稍后重试提示 |
| 500 | 服务端异常 | 记录错误并返回通用错误说明 |

### 前后端接口调用约定

1. 前端应将接口调用与页面状态分离，避免单个请求失败导致整个页面不可用；
2. 列表类接口应按稳定结构返回数组，即使无数据也应返回空数组；
3. 地图瓦片请求失败时，前端应尽量保持已有图层状态；
4. 搜索接口应限制结果数量，优先返回可被直接定位的节点和路线；
5. 接口新增字段应保持向后兼容，不应删除前端已使用字段；
6. 涉及外部来源链接的字段应在前端以安全方式打开；
7. 面向产业化复用的接口应优先保持响应字段稳定，确需调整时应通过版本号或兼容字段过渡。
