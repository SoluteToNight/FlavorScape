系统功能设计

本章对“寻味地理”风味博物志 WebGIS 系统的功能划分、模块结构和关键功能进行设计说明。系统功能围绕地图探索、风味基因库、时空叙事馆、食材传播、首页、关于与方法说明、通用能力和关键技术支撑展开。功能设计的重点不是单纯罗列页面，而是说明每个模块如何共同服务“风味知识的地图化表达”和后续专题化内容复用。

系统划分

系统按用户可见功能和后台支撑能力划分为以下模块：

模块功能定位主要用户操作
首页模块系统入口和主题识别查看系统主题，进入地图探索
地图探索模块核心 WebGIS 交互浏览地图，选择节点、路线、生态区，查看详情
风味基因库模块结构化风味浏览筛选风味节点，比较不同地域风味
时空叙事馆模块线性叙事阅读阅读章节，理解传播事件背景
食材传播模块专题化传播叙事选择食材，播放传播时间线，查看路径节点和历史说明
关于与方法说明模块来源和方法展示查看数据来源、方法框架和数据生产说明
异常页面模块页面异常处理返回首页或地图页
通用功能模块跨页面能力搜索、状态管理、图表展示、图层控制

系统不设置复杂后台管理界面。基础数据管理、种子数据导入和后续批量数据扩展主要通过数据导入流程完成，界面侧重点放在公众浏览和文化解释。若后续进入产业化阶段，可在不影响公众端体验的前提下另行建设内容管理后台，用于来源审核、专题发布和数据版本控制。

地图探索模块

地图探索模块是系统核心模块，承担 WebGIS 展示、图层叠加、对象选择、搜索定位和详情解释等功能。

模块技术架构

地图探索模块由地图渲染、WebGL 数据图层、业务状态、详情展示和搜索交互组成。地图渲染负责底图、物理矢量和生态区边界展示；数据图层负责主地图中的传播路线、流光动画和风味节点展示；业务状态负责控制节点、路线、生态区三类对象的互斥选择；详情展示负责根据选择对象输出解释内容；搜索交互负责从关键词定位到地图对象。

模块功能结构

子系统功能说明
地图渲染子系统显示自然地形底图、海岸线、河流和生态区边界
传播路径子系统显示传播路线和动态流光效果
风味节点子系统显示风味节点、聚合气泡和节点选中状态
交互选择子系统支持节点、路线、生态区点击选择和清除选择
详情展示子系统根据当前选择显示节点、路线或生态区详情
搜索子系统支持关键词搜索风味节点和传播路径
图层控制子系统支持地图图层显示、隐藏和上下文强化

模块功能描述

地图渲染子系统

地图渲染子系统应支持自然地形底图、海岸线、河流、生态区边界、传播路线和风味节点的叠加显示。图层顺序应保持稳定，底图位于最底层，生态区边界作为自然空间框架，传播路线和风味节点作为主要交互对象。

地图应支持缩放、平移和视图复位。全球尺度下应突出风味传播路径的宏观方向，区域尺度下应突出风味节点和生态区之间的关系。

传播路径子系统

传播路径子系统用于展示食材、技法或味型在历史时空中的迁移方向。传播路径应包含路线类型、路线颜色、关键节点和路径几何。系统可通过路径线和流光动画表现传播的方向性和动态性。路径可视化应让用户明确看到“从哪里来、向哪里去”，不能只依赖镜头移动表达传播过程。

用户点击传播路径后，系统应选中对应路线，并在详情面板中展示路线名称、路线类型、关键节点和传播说明。选中路线时应清除节点和生态区选择。播放或高亮结束后，地图应能够回到稳定视角，避免用户在旋转或飞行后失去空间参照。

风味节点子系统

风味节点子系统用于展示地方菜肴成品。节点应包含城市或地域、菜品名称、主风味标签、颜色和坐标位置。高信息密度场景下，节点可采用聚合或层级展示方式降低视觉拥挤。

用户点击风味节点后，系统应选中对应节点，并在详情面板中展示风味雷达图、描述文本、食材标签和同类型风味关联。选中节点时应清除路线和生态区选择，并显示相关传播上下文。

生态区选择子系统

生态区选择子系统用于展示自然环境对风味形成的空间背景。用户点击生态区后，系统应选中对应生态区，并展示生态区名称、生物群系、生物地理界和环境说明。

选中生态区时应清除节点和路线选择，并强化生态区边界图层，使用户能够将风味节点置于自然环境框架中理解。

详情展示子系统

详情展示子系统根据选择对象类型动态切换内容。节点模式展示风味向量、食材构成、地方描述和同类型关联；路线模式展示传播路线说明和关键节点；生态区模式展示生态区属性和环境解释。

详情面板应支持关闭或折叠，避免遮挡地图主体。面板内容应采用清晰的信息层级，先展示名称和摘要，再展示图表、标签和解释文字。

搜索子系统

搜索子系统支持用户通过关键词查找风味节点和传播路线。搜索结果应展示对象名称、简要说明和颜色标识。用户点击搜索结果后，系统应跳转到地图页并选中对应对象。

搜索结果数量应进行限制，避免候选项过多影响操作。无结果时应给出清晰反馈。

图层控制子系统

图层控制子系统提供基础图层、生态区图层、传播路径图层和风味节点图层的显示控制。图层控制应与对象选择联动，例如选择节点或路线时显示传播路径，选择生态区时强化生态区边界。

图层控制主要服务于浏览和调试，不应干扰普通用户理解核心内容。

风味基因库模块

风味基因库模块用于结构化展示风味节点，适合用户在非地图场景下浏览和比较不同地域风味。

模块功能结构

功能说明
节点列表以卡片网格展示风味节点
分类筛选按风味类型或标签筛选
风味概览展示主风味标签和雷达图摘要
地图跳转点击卡片进入地图并选中对应节点

模块功能描述

风味基因库应展示食物名称、地域、所属生态区、风味标签和风味向量概览。用户可通过分类筛选快速查看辛辣、鲜甜、咸香、酸鲜等不同类型风味。

该模块应支持从列表浏览进入空间浏览。用户点击风味卡片后，系统应打开地图并定位到对应风味节点，从而形成“结构化比较—地图解释”的浏览路径。

风味基因库也是后续内容产品化的基础入口。对于文旅、课程或展陈场景，可以按城市、食物属、味型或传播主题筛选同一批风味节点，形成不同专题包，而不需要重新整理底层数据。

时空叙事馆模块

时空叙事馆模块用于以章节形式展示典型传播事件和饮食文化材料。

模块功能结构

功能说明
章节导航按章节或时间线切换内容
正文阅读展示标题、年代、正文和引用
路线关联将章节与传播路径建立关联
地图入口从章节进入地图查看空间对象

模块功能描述

时空叙事馆应以线性阅读方式组织内容，帮助用户理解某类食材、技法或味型的传播背景。章节内容应避免只展示数据库字段，而应以公众可读的方式解释历史材料和空间过程。

章节应与地图对象保持关联。用户阅读某一章节后，可以进入地图查看对应传播路径或相关风味节点，形成“文本叙事—空间验证”的体验。

食材传播模块

食材传播模块用于展示单一食材的起源、传入、扩散和地方化过程，是当前系统从“风味节点展示”走向“专题化传播叙事”的重要页面。该模块面向辣椒、香料、茶叶等具有明确历史传播线索的食材，以时间线和地图路径共同组织内容。

页面应支持食材选择、时间线播放、路径节点展示、右侧说明文本和地图镜头联动。路径线应作为主视觉对象持续存在，镜头运动只负责强调阶段变化，不能替代路线绘制。播放速度应照顾阅读节奏，用户暂停或切换食材后，页面应清楚展示当前节点、当前路径段和对应文献说明。

从后续发展看，食材传播模块可以沉淀为专题模板。每个专题只要遵循统一 JSON 结构，就可以复用同一套页面能力生成新的传播故事。该模块适合扩展到文旅展陈、研学课程和地方品牌内容中，但前提是每个路径节点都具备明确来源或置信度标注。

首页模块

首页模块承担系统识别和入口引导功能。

首页应展示系统名称、主题定位和进入地图探索的主入口。首页视觉应体现饮食文化与自然地理的关系，可通过背景动效、代表性风味卡片或传播意象帮助用户建立第一印象。

首页不应成为营销式长页面，其首要任务是让用户快速理解系统主题并进入核心功能。

关于与方法说明模块

关于与方法说明模块用于说明系统的数据来源、方法框架和数据生产原则。

页面应展示 Natural Earth、WWF TEOW、图片材料和饮食文化材料等数据来源，说明每类数据在系统中的用途。页面还应说明风味向量、生态区映射、传播事件整理、食材传播数据结构和 LLM 辅助数据生产的基本方法。

LLM 辅助数据生产应被描述为后续扩展方向，且必须经过人工审核和质量控制。关于页面还应帮助用户理解系统边界：本系统面向饮食文化解释和知识展示，不提供餐馆点评、交易或实时供应链服务。

异常页面模块

异常页面模块用于处理不存在的路由或页面访问错误。

异常页面应提供简洁提示和返回入口，引导用户回到首页或地图页。异常页面不应展示服务端错误堆栈、内部路径或敏感信息。

通用功能模块抽取

全局搜索

全局搜索应在主要页面中保持可用，支持风味节点和传播路径检索。搜索结果应可直接触发地图对象选择。后续可扩展食材传播专题检索，但需要保持结果类型清晰，避免节点、路线和专题混在一起造成理解困难。

状态管理

系统应统一管理风味节点、传播路径、生态区、图层显示、当前章节和食材传播播放状态等状态。节点、路线和生态区选择应保持互斥；食材传播播放状态应与普通地图选择状态隔离，避免专题播放影响用户返回地图后的常规浏览。

图表展示

图表展示包括风味雷达图和生态区信息图表。图表应随当前选中对象同步更新，并与文字解释共同构成信息表达。

数据加载与异常反馈

系统应在数据加载失败、搜索无结果、瓦片暂不可用等情况下提供明确反馈。错误反馈应面向用户可理解，不应暴露内部实现细节。

系统实现的几个关键技术

WebGIS 混合渲染技术

系统同时使用地图引擎和 WebGL 数据可视化能力。地图引擎负责底图和基础矢量图层，WebGL 数据图层负责主地图中的传播路径、动画和风味节点。两类图层需要在同一地图视图中保持坐标一致和交互一致。对于食材传播等专题页面，可根据稳定性和表现需求选择 MapLibre 原生图层或 WebGL 图层，但必须保证路径可见、状态可恢复。

对象关系空间数据库技术

系统使用对象关系数据库和空间扩展统一管理生态区边界、菜肴成品坐标和传播路线。空间索引用于提升图层输出和空间查询效率，关系型约束用于保证菜肴成品、食材和传播事件之间的引用完整性。

动态传播路径展示技术

传播路径通过路径线和流光粒子表达方向性。动态效果应服务于文化传播解释，不应影响地图浏览性能和对象选择准确性。

栅格瓦片缓存技术

自然地形底图按需生成瓦片，系统通过缓存机制减少重复渲染。瓦片服务异常时应返回可处理的响应，保证地图页面不会因单个瓦片失败而整体不可用。
