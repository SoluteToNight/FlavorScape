# 系统功能设计

本章主要根据系统建设目标和需求分析，对系统的功能模块进行划分，并分别对各模块的技术架构、功能结构和功能描述进行详细设计。

## 系统划分

"寻味地理"风味博物志 WebGIS 系统按照前后端分离架构，从前端页面功能和后端服务能力两个维度进行模块划分：

**前端功能模块（5 个页面）**

| 模块 | 视图文件 | 核心职责 |
| --- | --- | --- |
| 首页模块 | `HomeView.vue` | 品牌展示、粒子动画入口 |
| 地图探索模块 | `MapView.vue` | 核心 WebGIS：四层地图 + 联动选择 + 详情面板 |
| 风味基因库模块 | `LibraryView.vue` | 卡片网格浏览 + 分类筛选 |
| 时空叙事馆模块 | `NarrativeView.vue` | 章节阅读 + 路线动画 |
| 关于模块 | `AboutView.vue` | 数据来源说明 |

**后端服务模块（7 个 API 端点组）**

| 模块 | 路由文件 | 核心职责 |
| --- | --- | --- |
| 风味数据服务 | `routers/api.py` | 风味节点查询、全局搜索 |
| 传播路线服务 | `routers/api.py` | 路线数据查询 |
| 叙事章节服务 | `routers/api.py` | 章节内容查询 |
| 数据来源服务 | `routers/api.py` | 元数据查询 |

| 矢量瓦片服务 | `routers/tiles.py` | 生态区 MVT 瓦片 + GeoJSON 图层 |
| 数据管线模块 | LLM 脚本（独立） | 风味向量提取 + 传播事件录入 |

## 地图探索模块

地图探索模块（MapView.vue）是系统的核心交互界面，承担四层地图渲染、用户交互和详情展示的职责。

### 模块技术架构

地图模块的技术栈层次如下：

```
MapView.vue (Vue 3 Composition API)
│
├── MapLibre GL JS (底图渲染引擎)
│   ├── style layers: bg / hyp (raster) / coastline / rivers / ecoregions
│   └── projection morphing (globe → mercator)
│
├── Deck.gl (WebGL 数据图层框架)
│   ├── MapboxOverlay (interleaved: true)
│   ├── PathLayer (L2 传播路径暗线)
│   ├── TripsLayer (L2 流光粒子动画)
│   └── ScatterplotLayer (L3 节点光晕)
│
├── ECharts (图表渲染)
│   ├── FlavorRadar.vue (六维风味雷达图)
│   └── MapInspector.vue (气候双轴曲线图)
│
├── Pinia Store (app.js)
│   ├── selectedNode / selectedRoute / selectedEcozone (互斥选择)
│   └── layerVisibility / layerEnabled (图层控制)
│
└── HTML overlays (屏幕空间节点气泡 + CSS 定位工具面板)
```

### 模块功能结构

地图模块的功能层次如下：

```
地图探索模块 (MapView)
├── 地图渲染子系统
│   ├── L0 物理底图 (HYP 栅格 + 海岸线/河流)
│   ├── L1 生态区边界 (TEOW 827 多边形)
│   ├── L2 传播路径 (5 条路线 PathLayer + TripsLayer)
│   ├── L3 风味节点 (17 节点 ScatterplotLayer + HTML 气泡)
│   └── 投影自动切换 (globe/morph/mercator)
├── 交互子系统
│   ├── 缩放与漫游
│   ├── 节点/路线/生态区点击选择
│   ├── 悬停路线 tooltip
│   ├── 清除选择 (Escape 键 / 点击地图空白)
│   └── 气泡聚类 (低缩放级别自动聚合)
├── 详情展示子系统
│   ├── MapInspector 右侧面板
│   │   ├── 节点模式 (FlavorRadar + 食材列表)
│   │   ├── 路线模式 (路线信息 + 途经节点)
│   │   └── 生态区模式 (气候图表 + 生物群系)
│   └── 图例面板 (左上角固定)
├── 搜索子系统
│   └── OmniSearch 全局搜索 (模糊匹配节点/路线)
└── 调试子系统
    └── DevTools 面板 (图层开关 + 状态读数)
```

### 模块功能描述

#### 地图渲染子系统

**L0 物理底图**：由 Natural Earth HYP 栅格瓦片层（`hyp`）和三个矢量图层（`coastline`、`rivers`、`land`）组成。HYP 栅格瓦片通过 `/tiles/raster/{z}/{x}/{y}.png` 接口获取，支持 0~10 级缩放。海岸线、河流线条以极浅灰色半透明细线渲染，不抢夺焦点。陆地面层用于在 globe 投影模式下提供极地以外的填充基准。

**L1 生态区边界**：827 个 TEOW 生态区 MultiPolygon 以 MapLibre geojson 图层渲染。默认状态下以极细浅褐色实线（1 px）勾勒边界，fill-opacity = 0（完全透明），仅在用户选中某生态区时强化线宽并微亮填充以示强调。

**L2 传播路径**：采用 Deck.gl 双图层叠加策略——

- **PathLayer**（底层暗线）：以低透明度暗灰色细线绘制路径静态轨道，虚线表越洋路线、实线表陆路/内河路线；
- **TripsLayer**（上层流光粒子）：每条路线生成 3 个错峰粒子，沿路径循环滑动。粒子呈高亮光斑效果，运动方向即传播方向。在跨越 L1 生态区边界时短暂减速加亮，暗示环境阻力。动画循环周期为 2200 ms。

**L3 风味节点**：采用 Deck.gl + HTML 双层渲染策略——

- **ScatterplotLayer**（底层光晕）：在地表坐标正上方呈现微弱呼吸光晕（半透明径向渐变圆），光晕颜色取自节点的 `color_hex`；
- **HTML 气泡卡片**（上层标签）：通过 CSS `transform` 将 HTML 元素投影到屏幕空间，每个节点显示城市名、菜品缩略图、风味主标签。高缩放级别（zoom > 4）时独立显示，低缩放级别（zoom ≤ 4）时自动聚合成气泡簇，显示聚合数量。

**投影自动切换**：根据当前缩放级别在三种投影间平滑过渡——

| 缩放级别 | 投影模式 | 视觉特征 |
| --- | --- | --- |
| zoom < 2.2 | globe（球面） | 类地球仪视角，倾斜角强制归零，极地加遮罩 |
| 2.2 ≤ zoom ≤ 2.8 | morph（过渡） | 球面到平面的平滑过渡 |
| zoom > 2.8 | mercator（墨卡托） | 标准 Web 墨卡托平面投影 |

#### 交互子系统

**互斥选择机制**（Pinia Store 状态管理）：节点、路线、生态区三者互斥选择——选中新对象时自动清除前一个对象的选择状态。选择行为驱动地图行为和面板内容：

| 选择状态 | L2 可见 | L1 强化 | MapInspector 模式 | 地图行为 |
| --- | --- | --- | --- | --- |
| 选中节点 | 显示 | 否 | 节点模式（雷达图+食材） | flyTo 飞行至节点坐标 |
| 选中路线 | 显示 | 否 | 路线模式（路线信息） | 无自动飞行 |
| 选中生态区 | 隐藏 | 线宽+填充 | 生态区模式（气候图） | fitBounds 至生态区范围 |
| 无选择 | 隐藏 | 否 | 空状态提示 | 无 |

**清除选择**：用户可通过以下任一操作清除当前选择——按 Escape 键、点击地图空白区域、搜索后直接跳转。

**悬停交互**：鼠标悬停 L2 PathLayer 路径时，显示路线名称 tooltip。

#### 详情展示子系统

**MapInspector 右侧面板**：可折叠面板（宽度约 25%），根据选中对象类型动态切换显示内容：

- **节点模式**：菜品名称 + 城市/区域/生态区信息 → ECharts 六维风味雷达图（FlavorRadar） → 风味因果推演文案 → 食材标签列表 → 同属其他节点链接；
- **路线模式**：路线名称 + 类型（陆路/海路） + 传播媒介 + 途经关键节点；
- **生态区模式**：生态区中英文名 + 生物群系标签 + ECharts 气候双轴曲线图（温度 + 降水量）+ 面积信息。

**图例面板**：左上角固定面板，标注当前四层图层的视觉元素对应关系（底色块+图层名）。

#### 搜索子系统

OmniSearch 组件位于地图页左上方（非地图页居中），提供以下功能：

1. 输入关键词（中文/拼音） → 调用 `/api/search?q=` 接口；
2. 后端使用 PostgreSQL tsvector + ILIKE 回退进行模糊匹配，搜索范围覆盖风味节点（`dish_name`、`city_name`、`region_label`）和传播路线（`route_name`）；
3. 下拉列表展示搜索结果，每条结果附带类型标记（节点/路线/生态区）和颜色圆点；
4. 点击搜索结果 → 若在非地图页则先导航至 `/map`，然后选中对应对象并 flyTo 飞行。

#### 调试子系统

DevTools 面板（右上角可折叠）提供开发者调试功能：

1. **图层独立开关**：L0、L1、L2、L3 四层各一个 toggle，仅对开发者可见；
2. **状态读数**：实时显示当前缩放级别、投影模式、globe 视图状态（是/否）、节点数量、气泡簇数量。

## 风味基因库模块

### 模块技术架构

风味基因库模块（LibraryView.vue）是一个纯前端页面，通过 `/api/flavors` 接口获取全量风味节点数据，在前端进行分类筛选和卡片渲染。每个风味卡片内嵌一个轻量级 FlavorRadar 组件（ECharts 雷达图）。

### 模块功能结构

```
风味基因库模块 (LibraryView)
├── 分类筛选栏 (全部/辛辣/鲜甜/咸香/酸鲜)
├── 风味卡片网格
│   └── 单张卡片
│       ├── 城市名 + 菜品名
│       ├── 所属生态区
│       ├── 六维风味雷达图 (FlavorRadar 缩略图)
│       ├── 主风味标签
│       └── 点击跳转至地图
└── 响应式布局 (grid-cols)
```

### 模块功能描述

1. **分类筛选**：顶部水平筛选栏，5 个类别按钮（全部、辛辣、鲜甜、咸香、酸鲜），点击切换筛选条件。筛选基于 `category` 字段精确匹配，"全部"显示所有节点；
2. **卡片网格**：响应式网格布局（CSS Grid），根据视口宽度自适应列数。每张卡片以毛玻璃面板呈现，底部有微弱阴影，悬停时上浮和阴影加深；
3. **雷达图缩略图**：每张卡片内嵌一个 140×140 px 的 FlavorRadar 组件，六维雷达图采用节点配色，去除坐标轴标签和装饰元素；
4. **点击跳转**：点击卡片 → Pinia Store 的 `selectNode()` action 选中对应节点 → Vue Router 导航至 `/map` → MapView 接收选中状态后 flyTo 飞行至节点坐标。

## 时空叙事馆模块

### 模块技术架构

时空叙事馆模块（NarrativeView.vue）通过 `/api/chapters` 和 `/api/routes` 接口获取数据，左侧渲染章节导航时间线，右侧渲染章节正文，顶部嵌入 Canvas 绘制的路径缩略动画。

### 模块功能结构

```
时空叙事馆模块 (NarrativeView)
├── 左侧章节导航 (时间线脊柱)
│   ├── 圆点标记 (当前章节高亮)
│   └── 点击切换章节
├── 右侧正文区
│   ├── 标题 + 年代标签
│   ├── Canvas 路径缩略动画
│   ├── 正文段落
│   ├── 文献引用 + 来源
│   └── "打开全地图自由探索"按钮
└── 底部进度点
```

### 模块功能描述

1. **章节导航**：左侧垂直时间线，以 CSS 绘制的"脊柱"连接各章节圆点。当前章节圆点高亮（琥珀黄色 + 放大），已读章节灰色，未读章节浅灰；
2. **路径缩略动画**：每章内容区顶部嵌入小型 Canvas，绘制与该章节关联的传播路线暗线动画（单粒子沿路径滑动，色彩与路线 `route_color` 一致）。动画循环播放；
3. **正文展示**：标题使用衬线体（Noto Serif SC），正文使用无衬线体（Noto Sans SC），行间距 1.8。章节末尾标注文献引用（`cite`）和来源（`source`）；
4. **探索入口**：每章末尾提供"打开全地图自由探索"按钮，点击后导航至 `/map`。

## 首页模块

### 模块功能描述

## 系统实现的几个关键技术

### MapLibre + Deck.gl 混合渲染技术

本系统实现了 MapLibre GL JS 和 Deck.gl 的混合渲染。核心挑战在于正确处理两个 WebGL 渲染管线的遮挡关系。解决方案是使用 Deck.gl 的 `MapboxOverlay` 组件，设置 `interleaved: true` 模式，使 Deck.gl 图层与 MapLibre 图层交替渲染——MapLibre 先渲染 L0（底图）和 L1（生态区），Deck.gl 随后渲染 L2（路径），MapLibre 再渲染标签层，最后 Deck.gl 渲染 L3（节点光晕）。HTML 气泡节点通过 CSS `transform: translate()` 从地理坐标投影到屏幕空间，独立于 WebGL 管线。

### 对象关系数据库技术

本系统利用 PostgreSQL 的对象关系特性实现灵活的混合数据模型：

1. **JSONB 半结构化字段**：风味向量（`genome_vector`）和气候属性（`climate`）以 JSONB 存储。JSONB 兼具 JSON 的灵活性和关系型数据库的查询能力——支持 GIN 索引加速路径查询（如 `genome_vector->>'la'`），支持部分更新（`jsonb_set`），且数据在写入时解析为二进制格式，查询效率高于纯文本 JSON；
2. **空间类型与 PostGIS 函数**：利用 PostGIS 的 GEOMETRY/GEOGRAPHY 类型原生存储空间数据，通过 ST_AsMVT 实时生成矢量瓦片，通过 ST_AsGeoJSON 输出前端可直接消费的 GeoJSON，通过 ST_Simplify 降低高缩放级别下的几何复杂度；
3. **关联表的额外属性**：配方连结构表（`recipe_link`）不仅记录 M:N 关系，还携带 `role`（食材角色）、`importance`（贡献度）、`is_native`（是否本地物产）等关系属性，这是对象关系模型的典型应用。

### Deck.gl TripsLayer 粒子动画技术

本系统使用 Deck.gl 的 TripsLayer 实现传播路径的流光粒子动画。关键实现细节：

1. **路径数据生成**：将后端返回的 GeoJSON LineString 坐标数组按顶点均匀采样为 200 个等距点，作为 TripsLayer 的路径数据；
2. **错峰策略**：每条路径生成 3 个粒子，起始时间偏移分别为 0、733、1466（ms），在 2200 ms 循环周期内均匀分布，避免同路径粒子视觉重叠；
3. **颜色与混合**：粒子颜色取自路线的 `route_color`，通过 additive blending（加性混合）在暗色底线上产生高亮光斑效果，流动方向即基因传播方向；
4. **非匀速运动**：粒子在路径平坦段匀速，在跨越 L1 生态区边界附近通过调整 time 映射函数实现短暂减速，暗示环境阻力。此功能通过给 TripsLayer 的 `getTimestamps` 传入非均匀时间戳数组实现。

### 栅格瓦片 LRU 缓存技术

瓦片渲染是本系统的主要性能瓶颈（涉及磁盘 I/O 和图像重采样）。缓存策略如下：

1. **缓存结构**：基于 Python `OrderedDict` 实现的线程安全 LRU（Least Recently Used）缓存，最大容量 512 条目。键为 `(z, x, y)` 元组，值为 PNG 字节流；
2. **线程安全**：通过 `threading.Lock` 保护所有读写操作，支持多线程并发访问；
3. **淘汰策略**：当缓存满时，自动淘汰最久未访问的条目。通过 `OrderedDict.move_to_end()` 在每次访问时将条目移至末尾，实现 O(1) 的 LRU 语义；
4. **命中率监控**：缓存内部维护 `hits` 和 `misses` 计数器，通过 `/tiles/status` 接口暴露命中率指标。
