# 界面设计

本章对“寻味地理”风味博物志 WebGIS 系统的 B/S 界面设计原则、页面布局、主要页面界面、响应式策略和交互反馈进行说明。系统界面面向公众用户，以自然地理地图为主要视觉对象，以风味数据和叙事内容为辅助解释，形成清晰、克制、可读的 WebGIS 使用体验。

## B/S 界面设计

### 设计原则

1. **地图优先**：地图是系统核心表达界面，页面控件和信息面板不应遮挡主要空间对象。
2. **公众可读**：界面文本应避免专业术语堆叠，必要术语应通过上下文解释，使非 GIS 用户能够理解。
3. **信息分层**：名称、摘要、图表、标签、来源和操作入口应按层级排列，避免信息同时竞争视觉焦点。
4. **交互明确**：可点击对象应具有悬停、选中和清除状态，用户能够判断当前操作对象。
5. **风格统一**：首页、地图页、风味基因库、时空叙事馆和关于页面应保持统一的色彩、字体、间距和按钮样式。
6. **轻量表达**：系统界面应区别于传统专业 GIS 工具台，避免大量工具栏、复杂表格和过密按钮。
7. **来源可查**：涉及数据来源、图片来源和方法说明的内容应提供清晰入口，增强可信度。

### 整体布局

系统采用浏览器单页应用布局。顶部或边缘区域提供全局导航和搜索入口，主体区域根据页面类型展示地图、卡片网格、叙事正文或方法说明。

地图页以全屏地图为主体，搜索、图例、图层控制和详情面板浮于地图之上。非地图页面采用内容容器布局，保证文字阅读宽度适中，卡片和章节信息清晰排列。

## 系统首页界面

首页是系统的第一入口，主要任务是建立系统主题识别并引导用户进入地图探索。

### 页面组成

| 区域 | 内容 |
| --- | --- |
| 背景区域 | 与风味传播、地理流动或自然地形相关的视觉背景 |
| 标题区域 | 系统名称“寻味地理”和风味博物志副标题 |
| 操作区域 | 进入地图探索的主按钮 |
| 内容预览 | 代表性风味节点或传播主题预览 |

### 设计要求

首页应在首屏清晰表达“饮食文化地理探索”的主题。主操作按钮应明显可见，用户无需阅读长篇说明即可进入地图页。代表性风味预览应服务于内容引导，不应喧宾夺主。

首页动效应轻量、平稳，不应影响页面加载和按钮点击。移动端首页应保证标题、按钮和预览内容不互相遮挡。

## 系统地图页界面

地图页是系统核心工作界面，承担地图浏览、对象选择、搜索定位和详情展示。

### 页面组成

| 区域 | 内容 |
| --- | --- |
| 地图主体 | 自然地形底图、物理矢量、生态区边界、传播路径、风味节点 |
| 搜索入口 | 支持关键词检索风味节点和传播路径 |
| 图例区域 | 说明底图、生态区、路线和节点等视觉符号 |
| 图层控制 | 控制主要图层显示状态 |
| 详情面板 | 显示当前选中节点、路线或生态区详情 |
| 导航入口 | 提供前往首页、风味基因库、时空叙事馆和关于页面的入口 |

### 地图主体设计

地图主体应占据页面主要空间。自然地形底图作为背景，色彩应保持柔和；生态区边界应清晰但不过度突出；传播路径应具有方向感；风味节点应具有较强识别度。

当用户缩放地图时，节点、路线和生态区边界应保持空间对齐。低缩放级别下应避免节点过度拥挤，高缩放级别下应展示更多节点信息。

### 详情面板设计

详情面板根据选中对象切换内容。

| 选中对象 | 面板内容 |
| --- | --- |
| 风味节点 | 菜品名称、地域、生态标签、风味雷达图、描述文本、食材标签、同类型关联 |
| 传播路径 | 路线名称、路线类型、关键节点、传播说明 |
| 生态区 | 生态区名称、生物群系、生物地理界、生态区编码和环境说明 |

详情面板应支持折叠或关闭。面板展开时应保证地图仍可浏览，移动端可采用底部抽屉式布局。

### 搜索与选择设计

搜索框应支持输入关键词并展示结果列表。结果项应包含标题、副标题和颜色提示。点击结果后，系统应进入地图并选中对应对象。

节点、路线和生态区选择状态应互斥。选中对象应有明显高亮；点击地图空白区域或使用清除操作后，应恢复普通浏览状态。

### 图层与图例设计

图例应简要说明主要视觉元素，避免使用长说明文字。图层控制应以开关或按钮形式呈现，使用户能够快速显示或隐藏底图、生态区、传播路径和风味节点。

图层控制区不应干扰普通浏览，可采用折叠形式或放置在次要位置。

## 风味基因库页界面

风味基因库页用于结构化浏览风味节点。

### 页面组成

| 区域 | 内容 |
| --- | --- |
| 标题区 | 页面名称和简要说明 |
| 筛选区 | 风味类型或标签筛选 |
| 卡片网格 | 风味节点卡片 |
| 卡片内容 | 菜品名称、地域、生态区、主风味标签和雷达图概览 |

### 设计要求

卡片应以统一尺寸和稳定间距排列，便于横向比较。筛选状态应清晰可见，用户能够判断当前列表范围。

卡片点击后应进入地图并定位到对应风味节点。卡片中的图表应作为概览，不应过度复杂；详细解释应放在地图详情面板中展示。

## 时空叙事馆页界面

时空叙事馆页用于线性阅读典型传播事件和历史材料。

### 页面组成

| 区域 | 内容 |
| --- | --- |
| 章节导航 | 章节列表或时间线 |
| 正文区域 | 标题、年代标签、正文和引用来源 |
| 路线提示 | 与章节关联的传播路线视觉提示 |
| 地图入口 | 跳转至地图查看相关路径或节点 |

### 设计要求

章节导航应帮助用户理解内容顺序。正文区域应保持适合阅读的宽度和行距，引用来源应与正文区分但保持可见。

叙事馆不应将地图交互完整复制到页面内，而应通过简要路线提示和地图入口引导用户进入地图页查看空间细节。

## 关于页界面

关于页用于说明系统主题、数据来源和方法框架。

### 页面组成

| 区域 | 内容 |
| --- | --- |
| 项目说明 | 系统定位和建设目标 |
| 数据来源 | 基础地图、生态区、图片和文本材料来源 |
| 方法说明 | 风味向量、生态区映射、传播事件整理和 LLM 辅助流程 |
| 引用入口 | 外部来源链接 |

### 设计要求

关于页应以说明性内容为主，结构清晰、段落简洁。数据来源可采用列表或卡片形式展示，便于用户快速了解每类数据的用途。

LLM 辅助数据生产应说明为后续扩展机制，并明确人工审核和来源核对要求。

## 异常页面界面

异常页面用于处理不存在的页面或错误访问。

异常页面应包含简短提示、返回首页入口和返回地图入口。页面风格应与系统整体一致，不应展示内部错误堆栈、服务端路径或调试信息。

## 响应式布局策略

### 桌面端

桌面端优先保证地图浏览效率。地图页可采用左侧搜索、中央地图、右侧详情面板的布局；风味基因库可采用多列卡片网格；叙事馆可采用章节导航与正文并列布局。

### 平板端

平板端应减少固定面板宽度，允许详情面板折叠。卡片网格列数应随屏幕宽度减少，搜索结果和导航菜单应保持可触控。

### 移动端

移动端地图页应将详情面板转换为底部抽屉或覆盖式面板，搜索和图层控制应收纳为轻量入口。首页标题和主按钮应保持首屏可见。风味基因库应采用单列或双列卡片，叙事馆应以纵向阅读为主。

### 可访问性与可读性

界面应保证文字与背景具有足够对比度。交互控件应具备可点击区域，移动端按钮不应过小。颜色不应作为唯一状态提示，选中、禁用和错误状态应配合文字或形状变化。

## 界面反馈设计

1. 数据加载时应展示轻量加载状态；
2. 搜索无结果时应提供明确提示；
3. 地图对象选中后应有高亮或面板切换反馈；
4. 接口失败时应保留页面基本结构，并提示用户稍后重试；
5. 页面切换应保持平稳，不应造成明显闪烁；
6. 动画效果应服务于风味传播表达，不应影响阅读和点击操作。
