# 前端项目交接文档 - 2026-06-02

## 🛠️ 项目当前整体架构

项目目前是一个基于地理信息（GIS）的空间叙事与可视化生成系统。整体采用**前后端分离**架构，前端负责多维度的视觉呈现与海报生成，后端负责提供 GIS 数据存证、底层瓦片代理及食材参数接口。  

Plaintext

```
📁 前端 (Vue 3 + Vite)  ──[开发代理]──>  📁 后端服务矩阵
                                        ├── FastAPI (GIS核心/数据中心)
                                        └── Express (静态托管/代理网关)
```

## 💻 前端技术栈与核心依赖

- **核心框架**：Vue 3（组合式 API）、Vue Router（路由驱动）、Pinia（全局状态管理）。  
- **空间地理渲染**：
  - `MapLibre GL`：用于大屏的高清底图与矢量图层渲染。  
  - `Deck.gl`：用于渲染高级动态路径（TripsLayer）和散点（ScatterplotLayer）。  
  - `d3-geo`：用于商业营销页面的**静态矢量地图投影**，脱离 WebGL 依赖，保证海报级轻量渲染。
- **图表与图形**：`ECharts`（用于传播图谱及硬核科学品质看板）。  
- **图像导出**：`html2canvas`（用于全静态多端画布的 3x 印刷级 PNG 压制导出）。

## ⚙️ 商业营销页 (`MarketingView`) 的设计机制

该页面是本次重构的重点，彻底抛弃了在海报中初始化 WebGL Canvas 的低效做法，改用 **Schema 驱动的主题与布局矩阵**：

1. **数据同源与全局联动**：食材数据（汉源花椒/五常大米）及当前选中的溯源节点通过 Pinia（`src/stores/app.js`）跨页面共享，用户在切换功能块时数据保持同步。  
2. **布局与骨相分离**：通过 `activeTheme.layout`（如 `standard`, `split`, `grid`, `immersive`）动态修改 CSS 类名，实现了同一个食材数据在“自然画册”、“东方文人”、“风味蓝图”和“沉浸纪实”四种截然不同的视觉排版间无缝闪切。
3. **自适应地图避让**：底图直接读取轻量级省级行政区划 GeoJSON (`china.json`)，利用 `d3-geo` 进行屏幕等比投影。针对黑龙江、新疆等边缘省份，注记引线会自动触发象限判断并向内折叠，防止标签冲出海报视口。
