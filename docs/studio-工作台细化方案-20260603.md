## 1. 定位

**核心定位**（经过讨论确认）：

> `/studio` 是一个**空间数据 × 用户创造力的综合内容产出平台**。
> GIS 空间数据是不可变的底板，用户在此之上编辑文案、选择风格、配置输出，完成从「空间事实」到「品牌资产」的创作过程。

**关键认知**：
- **产品 JSON 提供起点**（空间事实 + 建议文案），用户不需要从零开始
- **Studio 是画布**（编辑面板 + 实时预览），用户参与设计而非被动浏览
- **导出是产出**（合并空间底板 + 用户创作 = 最终产物）

**与原有页面的关系**：
- 原有页面（`/brand`、`/marketing`、`/archive`、`/spread`）**完全不动**，其他成员继续调优
- Studio 的渲染逻辑**复制**原有页面的核心渲染区到新文件（如 `StudioPoster.vue`），共享 `StaticGeoMap.vue` 等底层组件
- 后续渲染逻辑改进，需要手动同步原始页面和 Studio 拷贝

## 2. 核心架构决策

### 2.1 数据架构：空间底板 + 创作层

```
产品 JSON（只读）                    Studio 项目状态（Pinia）      导出产物
──────────────────                  ──────────────────────      ────────
空间底板（不可变）                   用户编辑覆盖层                合并渲染
  ├ 坐标、气候、检测报告              ├ 修改后的标题               → 海报 PNG
  ├ 供应链节点位置                   ├ 选择的海报风格             → 白皮书 PDF
  └ 风味指纹数值                    ├ 可见节点清单              → 大屏嵌入代码
                                   └ 自定义主图
创作默认值（可编辑起点）
  ├ 建议标题文案
  ├ 推荐风格
  └ 预设叙事文本
```

每个模块的 JSON 拆为 `spatial`（不可变 GIS 事实）和 `creative`（带编辑元信息的默认值）两层。

详见：`docs/product-cases-schema设计.md`

### 2.2 组织模式：项目中心制

| 维度 | 决策 |
|------|------|
| 顶层实体 | **项目（Project）** |
| 默认行为 | 一个项目绑定一个产品，生成一套资产包 |
| 扩展能力 | 结构上允许一个项目绑定多个产品（品类线场景） |
| 配置持久化 | 第一阶段存 Pinia store（会话级），后续可接 localStorage 或后端 |
| 数据模型 | 项目状态 = { 产品引用, 各模块创作覆盖值, 产出物记录 } |

### 2.3 组件策略：复制渲染逻辑，不动原文件

不抽取共享组件。而是将 MarketingView / ArchiveView / GeoAtlasView 的渲染区**复制**到新文件：

```
MarketingView.vue ← 完全不动           StudioPoster.vue ← 新建，复制渲染逻辑
  ├ 内联产品数据                          ├ 接收 spatialData + creativeData props
  ├ 海报渲染区（内联）                     ├ 引用 StaticGeoMap（共享）
  └ 导出逻辑                              └ 导出逻辑

ArchiveView.vue ← 完全不动             StudioArchive.vue ← 新建
GeoAtlasView.vue ← 完全不动            StudioDisplay.vue ← 新建
```

共享的底层组件（`StaticGeoMap.vue`、`FlavorRadar.vue`）继续复用。

### 2.5 html2canvas 兼容性

| 渲染组件 | 技术 | html2canvas 兼容 | 导出方案 |

| `StudioArchive.vue` | ECharts + FlavorRadar | ✅ 完全兼容 | html2canvas → PNG（暂不含 MapLibre 地图） |

## 4. 数据流与状态管理

### 4.1 Pinia Store：`src/stores/studio.js`

使用 Composition API 风格 + `reactive()`（深嵌套自动响应）：

```js
import { defineStore } from 'pinia'
import { reactive, computed } from 'vue'
import { useProductCases } from '../composables/useProductCases'

export const useStudioStore = defineStore('studio', () => {
  const { getById } = useProductCases()

  // ── 项目列表（用 reactive 保证深嵌套响应）──
  const projects = reactive({})        // { [projectId]: ProjectState }
  const projectOrder = reactive([])    // 项目 ID 顺序列表
  let nextId = 1

  // ── 当前活跃项目 ──
  const activeProjectId = reactive({ value: null })

  // ProjectState 结构：
  // {
  //   id: 1,
  //   name: '花椒·电商方案',
  //   productId: 'hanyuan-pepper',
  //   activeOutput: 'poster',       // 'poster' | 'archive' | 'display' | 'spread'
  //   marketing: {
  //     desc: null,                 // null = 使用 product JSON 默认值
  //     poeticLine: null,
  //     narrative: null,
  //     theme: 'nature',            // 默认 nature
  //     customImage: null,
  //     copy_xiaohongshu: null,
  //     copy_ecommerce: null,
  //   },
  //   archive: {
  //     flavorSummary: null,
  //     originDesc: null,
  //   },
  //   display: {
  //     visibleNodeIds: null,       // null = 使用默认
  //     cameraMode: 'tour',
  //     stageStories: {},           // { origin: '用户改的文本', ... }
  //   },
  //   spread: {
  //     storySummary: null,
  //     brandStory: null,
  //   },
  //   outputStatus: {
  //     poster: 'pending',          // 'pending' | 'edited' | 'skipped'
  //     archive: 'pending',
  //     display: 'pending',
  //     spread: 'skipped',
  //   },
  // }

  // ── 计算：合并后的渲染数据 ──
  const mergedMarketingData = computed(() => {
    const p = activeProject.value
    if (!p) return null
    const product = getById(p.productId)
    if (!product) return null
    return {
      name: product.name,
      heroImage: p.marketing.customImage || product.heroImage,
      spatial: product.marketing.spatial,
      creative: {
        desc:       p.marketing.desc       ?? product.marketing.creative.desc.default,
        poeticLine: p.marketing.poeticLine ?? product.marketing.creative.poeticLine.default,
        narrative:  p.marketing.narrative  ?? product.marketing.creative.narrative.default,
        theme:      p.marketing.theme      ?? product.marketing.creative.theme.default,
        copy: {
          xiaohongshu: p.marketing.copy_xiaohongshu ?? product.marketing.creative.copy.xiaohongshu.default,
          ecommerce:   p.marketing.copy_ecommerce   ?? product.marketing.creative.copy.ecommerce.default,
        },
      },
    }
  })

  // 同理：mergedArchiveData, mergedDisplayData, mergedSpreadData

  // ── 计算：活跃项目 ──
  const activeProject = computed(() => {
    const id = activeProjectId.value
    return id ? projects[id] : null
  })

  const projectList = computed(() => {
    return projectOrder.map(id => ({ id, ...projects[id] }))
  })

  // ── 动作 ──
  function createProject(productId, name) {
    const id = nextId++
    const project = {
      id,
      name: name || `未命名项目 ${id}`,
      productId,
      activeOutput: 'poster',
      marketing:    { desc: null, poeticLine: null, narrative: null, theme: 'nature', customImage: null, copy_xiaohongshu: null, copy_ecommerce: null },
      archive:      { flavorSummary: null, originDesc: null },
      display:      { visibleNodeIds: null, cameraMode: 'tour', stageStories: {} },
      spread:       { storySummary: null, brandStory: null },
      outputStatus: { poster: 'pending', archive: 'pending', display: 'pending', spread: 'skipped' },
    }
    projects[id] = project
    projectOrder.push(id)
    activeProjectId.value = id
    return id
  }

  function updateCreative(module, field, value) {
    const p = activeProject.value
    if (!p) return
    if (module === 'marketing') {
      p.marketing[field] = value
      if (p.outputStatus.poster === 'pending') p.outputStatus.poster = 'edited'
    }
    // 同理 archive, display, spread
  }

  function resetToDefault(module, field) {
    updateCreative(module, field, null)
  }

  function setActiveOutput(type) {
    if (activeProject.value) activeProject.value.activeOutput = type
  }

  function setOutputStatus(type, status) {
    if (activeProject.value) activeProject.value.outputStatus[type] = status
  }

  function switchProject(id) {
    activeProjectId.value = id
  }

  async function exportAsset(type) {
    // 根据 type 调用对应的导出逻辑
    // 海报：html2canvas → PNG
    // 白皮书：html2canvas → PNG（不含地图）
  }

  async function exportAll() {
    const p = activeProject.value
    if (!p) return
    for (const type of ['poster', 'archive', 'display', 'spread']) {
      if (p.outputStatus[type] !== 'skipped') {
        await exportAsset(type)
      }
    }
  }

  function hasAnyProject() {
    return projectOrder.length > 0
  }

  return {
    projects, projectOrder, activeProjectId,
    activeProject, projectList,
    mergedMarketingData, mergedArchiveData, mergedDisplayData, mergedSpreadData,
    createProject, updateCreative, resetToDefault,
    setActiveOutput, setOutputStatus, switchProject,
    exportAsset, exportAll, hasAnyProject,
  }
})
```

### 4.2 ImportFlow 规范

**触发条件**：用户在没有项目时，点击 `EmptyGuide.vue` 中的产品卡片，或点击「导入」按钮。

**动画流程**（简单 CSS 文字动画，无后端操作）：

```
┌──────────────────────────────────────────┐
│                                          │
│           ◉ 正在识别产品名称              │  ← 第 1 步：0.8s，显示产品名
│           ○ 正在提取产地信息              │
│           ○ 正在匹配地理节点              │
│           ○ 正在生成空间资产              │
│                                          │
│              [ 跳过 ]                     │
└──────────────────────────────────────────┘
```

- 4 个步骤依次高亮，每步约 0.8 秒
- 每步高亮时，显示当前处理的内容（如"识别产品名称：漢源花椒"）
- 动画总时长约 3.2 秒
- 点击「跳过」或动画自然结束 → 项目创建 → 进入编辑模式
- **这是一个模拟动画**，Phase 1 不做真实文件解析

**Phase 1 占位入口**：`EmptyGuide.vue` 底部有一个「或点击导入已有品牌资料 →」链接，点击后显示「此功能即将上线，当前为演示模式」提示，但可通过产品卡片正常创建项目。

### 4.3 产品数据 Composable：`src/composables/useProductCases.js`

```js
import { ref } from 'vue'

const cases = ref([])
const loading = ref(false)
const error = ref(null)

export function useProductCases() {
  async function loadAll() {
    if (cases.value.length) return
    loading.value = true
    error.value = null
    try {
      // Phase 1: 用 Vite 的 import.meta.glob 静态加载
      const modules = import.meta.glob('@/data/product_cases/*.json', { eager: true })
      cases.value = Object.values(modules).map(m => m.default)
    } catch (e) {
      error.value = e.message
      console.error('[useProductCases]', e)
    } finally {
      loading.value = false
    }
  }

  function getById(id) {
    return cases.value.find(c => c.id === id) || null
  }

  function getByCategory(cat) {
    return cases.value.filter(c => c.category === cat)
  }

  return { cases, loading, error, loadAll, getById, getByCategory }
}
```

## 8. 风险与注意事项

1. **渲染逻辑拷贝的同步** — StudioPoster / StudioArchive / StudioDisplay 与原始页面各有一份渲染逻辑。MarketingView 改进后，需要手动同步到 StudioPoster。建议在代码注释中标注"同步自 MarketingView Lx-Ly"。

2. **html2canvas + WebGL** — 白皮书和大屏导出不含 MapLibre 地图。第二阶段改用 `getCanvas().toDataURL()`。

3. **Pinia reactive() 使用注意** — `projects` 用 `reactive({})`，不能整体替换（`projects = {}` 会丢失响应式），只能改属性。

4. **编辑状态丢失** — Phase 1 仅存 Pinia（刷新丢失），`beforeunload` 事件提示用户「有未保存的更改」。

5. **import.meta.glob 在生产环境** — Vite 构建时会将 JSON 打包进 bundle。如果产品 JSON 数量增长，后续需要改为 `fetch()` 动态加载。

## 附录：讨论过程摘要

| 讨论点 | 初始方向 | 最终决策 |
|--------|----------|----------|
| 组织模式 | 产品中心 vs 项目中心 | **项目中心制** |
| 组件策略 | 抽取共享组件 | **复制渲染逻辑，不动原文件** |
| 产品数据 | 内联 vs 共享数据层 | **product_cases JSON，spatial+creative 双层** |
| 数据架构 | 固定内容 | **空间底板（不可变）+ 创作默认值（可编辑）** |
| Studio 角色 | 内容预览 | **创作画布**：编辑面板 → 实时预览 → 导出 |
| 布局模式 | 侧边栏+画布 | **编辑面板 + 实时画布 + 产出物清单 + 空间底板** |
| 空状态 | 未设计 | **引导卡片 + 产品选择器 → ImportFlow** |
| ImportFlow | 概念提及 | **4 步 CSS 文字动画（模拟），Phase 1 不做真实导入** |
| Pinia 响应性 | ref({}) | **reactive() 保证深嵌套响应** |
| html2canvas | 假设全兼容 | **SVG 兼容，WebGL 不兼容；MAP 降级处理** |
| 验收标准 | 无 | **每个任务补充具体完成定义** |
| Navbar | 新增入口 | 「创作工作台」放第一位，首页 CTA 指向 studio |
