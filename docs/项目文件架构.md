# 项目文件架构

本文档是 FlavorScape 的简明目录说明，用于快速判断哪些文件参与网站运行，哪些只是开发、设计或交接资料。

## 核心运行文件

- `src/`：Vue 前端源码。页面、组件、路由、状态和全局样式都在这里。
- `public/`：前端静态资源。Vite 会按原路径直接发布，运行页面引用的公开图片应放这里。
- `backend/`：FastAPI 后端。`backend/main.py` 是服务入口，`backend/routers/` 放 API 和瓦片路由，`backend/db/` 负责 PostGIS 连接、迁移和查询，`backend/data/` 放后端展示元数据。
- `api/`：Express 生产模式下的轻量硬编码 API 数据源。
- `data/`：运行所需业务/GIS 数据。`data/ingredient/*.json` 被食材传播页面读取，GIS 栅格/矢量数据由 FastAPI 启动流程使用。
- `index.html`、`vite.config.js`、`package.json`、`package-lock.json`：前端构建和开发服务配置。
- `server.js`：Express 服务入口，开发时和 Vite 一起由 `npm run dev` 启动，生产时服务 `dist/`。
- `start.ps1`、`start.sh`：一键启动脚本。Windows 优先使用 `.\start.ps1`。
- `pyproject.toml`、`.python-version`、`backend/requirements.txt`：Python 环境和依赖配置。
- `.env.example`：环境变量示例。真实 `.env` 不提交。

## 前端结构

- `src/main.js`：前端应用入口。创建 Vue 应用，挂载 Pinia 状态、Vue Router，并引入全局样式。
- `src/App.vue`：应用外壳。负责承载导航和当前路由页面。
- `src/router/index.js`：前端路由表。把 URL 路径映射到 `src/views/` 下的页面组件。
- `src/stores/app.js`：Pinia 全局状态。主要维护地图/传播相关选择状态，节点、路线、生态区选择互斥。
- `src/styles/global.css`：全局基础样式、颜色变量、布局底色和通用视觉规则。

### 页面目录

- `src/views/HomeView.vue`：首页/总览入口。
- `src/views/MapView.vue`：WebGIS 地图页，使用 MapLibre、Deck.gl、后端 `/tiles/*` 和 `/api/*` 数据。
- `src/views/IngredientSpreadView.vue`：食材传播图谱页，读取 `data/ingredient/*.json` 对应的数据。
- `src/views/GeoAtlasView.vue`：智慧大屏/地理志页面，聚焦沉浸式地图叙事、产品节点巡航和空间证据展示。
- `src/views/MarketingView.vue`：营销海报与商业展示物料页面。
- `src/views/ArchiveView.vue`：实证白皮书/证明材料展示页面。
- `src/views/LibraryView.vue`：资料库/内容集合页面。
- `src/views/NarrativeView.vue`：叙事展示页面。
- `src/views/AboutView.vue`：项目介绍页面。

### 组件目录

- `src/components/Navbar.vue`：全站导航组件。
- `src/components/NavDot.vue`：导航或页面定位用的小型交互组件。
- `src/components/MapInspector.vue`：地图信息检查面板，用于展示当前选中的节点、路线或生态区信息。
- `src/components/FlavorRadar.vue`：风味雷达图组件。
- `src/components/OmniSearch.vue`：综合搜索组件。
- `src/components/StaticGeoMap.vue`：静态地图/地理可视化组件。

### 前端资源

- `src/assets/`：被源码直接 import 的前端资源，例如 `src/assets/china.json`。这类资源会被 Vite 打包处理。
- `public/ingredients/`：可通过固定 URL 访问的食材图片，例如 `/ingredients/pepper-realistic.png`。适合页面直接引用的公开静态图。
- `docs/assets/reference-pictures/`：设计参考图，只用于开发参考，不应被运行页面直接引用。
- `dev/screenshots/`：验证截图和调试截图，不参与打包。

### 前后端调用关系

- 开发环境浏览器访问 Vite `5173`。
- Vite 将 `/api/*` 和 `/tiles/*` 代理到 FastAPI `8001`。
- `MapView.vue` 依赖 FastAPI 地图瓦片和地理数据；如果只启动前端，地图相关能力可能不完整。
- 生产模式下 `server.js` 服务 `dist/` 和 Express 自带 `/api/*`，但不代理 `/tiles/*`，完整 WebGIS 仍需要单独 FastAPI 后端或反向代理。

## 开发和文档文件

- `docs/`：项目文档、交接记录、总体设计材料。
- `docs/assets/illustrations/`：总体设计、汇报文档使用的 SVG 插图，不参与网站运行。
- `docs/assets/reference-pictures/`：页面和视觉优化参考图，不参与网站运行。
- `dev/screenshots/`：调试、Playwright、人工验收截图。
- `dev/logs/`：本地验证和启动日志。
- `dev/artifacts/`：本地临时大文件或归档包，例如 `data.zip`。
- `tools/`：开发辅助脚本，主要用于生成文档配图或设计图。
- `CLAUDE.md`、`AGENTS.md`、`README.md`：协作和启动说明。

## 启动关系

开发时通常是三进程：

1. FastAPI 运行在 `127.0.0.1:8001`。
2. `npm run dev` 启动 Express `3001` 和 Vite `5173`。
3. 浏览器访问 Vite，`/api/*` 和 `/tiles/*` 由 Vite 代理到 FastAPI。

生产静态构建使用：

```powershell
npm run build
npm run start
```

注意：`server.js` 只服务 `dist/` 和自己的 `/api/*`，不代理 FastAPI 的 `/tiles/*`。
