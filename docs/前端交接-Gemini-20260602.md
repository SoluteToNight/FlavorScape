# 前端交接给 Gemini - 2026-06-02

## 当前进度概览

项目当前是 Vue 3 + Vite 前端，配合 FastAPI GIS 后端与 Express 静态/代理服务。前端已经从单纯 WebGIS 展示转向“地方食材品牌故事地图 + 轻量溯源可视化生成器”的产品化原型。

当前主要页面：

- `/`：入口页。
- `/brand`：产品生成器/品牌故事页，目前是前端视觉改造重点。
- `/spread`：食材历史传播图谱，读取食材传播数据并在 MapLibre 上展示路线、节点、时间线。
- `/map`：原 WebGIS 风味底图，使用 MapLibre + Deck.gl 渲染 raster/vector/route/node 层。
- `/library`、`/narrative`、`/about`：辅助页面。

## 启动和验证

推荐 Windows 一键启动：

```powershell
.\start.ps1
```

手动启动：

```powershell
.venv/Scripts/python.exe -m uvicorn backend.main:app --host 127.0.0.1 --port 8001 --reload
npm run dev
```

访问地址：

- 前端 Vite：`http://localhost:5173`
- Express：`http://localhost:3001`
- FastAPI：`http://localhost:8001`
- FastAPI 文档：`http://localhost:8001/docs`

前端构建验证：

```powershell
npm run build
```

注意：`package.json` 没有 lint/test 脚本，当前主要用 `npm run build`、`/health`、`/tiles/status` 和页面截图做 smoke check。

## 前端技术结构

入口：

- `src/main.js`：挂载 Vue、Pinia、Router、全局 CSS。
- `src/App.vue`：固定 `Navbar` + `RouterView`，页面切换用 transition。
- `src/router/index.js`：定义全部前端路由。
- `src/stores/app.js`：全局 Pinia 状态，尤其是地图选择状态和图层开关。

关键依赖：

- Vue 3、Vue Router、Pinia
- MapLibre GL
- Deck.gl
- ECharts
- html2canvas
- Vite

Vite 开发代理在 `vite.config.js`：

- `/api/*` -> `http://localhost:8001`
- `/tiles/*` -> `http://localhost:8001`

Express 的 `server.js` 当前也会代理 `/api` 和 `/tiles` 到 FastAPI；生产模式托管 `dist/`。

## 重点页面说明

### `/brand` - `src/views/BrandStoryView.vue`

这是当前最重要的前端页面，也是 Gemini 优先接手对象。

页面定位：把地方食材的产地、加工、仓储、市场、餐饮应用等信息，生成可展示、可导出的品牌故事视觉产品。

已具备能力：

- 多视图切换，包含地理志、品牌公示、产业链、风物档案等形态。
- 使用真实 MapLibre 地图作为溯源证据底图。
- 节点和路线按当前产品数据动态生成。
- html2canvas 导出 PNG。
- 当前正在做 v4 方向的视觉打磨：减少厚重面板，突出地图、路线、节点和注释线。

待继续完善：

- 检查 v4 CSS 覆盖在不同视口下是否有文字溢出、浮层遮挡、地图标注错位。
- 把当前散落在单文件内的大量产品数据、模板数据、视图状态逐步模块化。
- 增加更多真实食材数据、图片素材、证据来源字段。
- 导出 PNG 需要重新截图验证，尤其是 MapLibre canvas 和浮层是否完整进入导出图。

### `/spread` - `src/views/IngredientSpreadView.vue`

食材传播图谱页面。当前结构比较完整，包含：

- 多食材总览。
- 单食材传播时间线。
- MapLibre 地图节点/路线。
- 右侧食材选择和历史事件列表。

接手建议：

- 优先保证数据结构稳定，不要随意改 `ingredient_id`、事件坐标、时间线字段。
- 如果优化视觉，先做小范围 CSS 调整，避免影响地图投影和事件定位逻辑。

### `/map` - `src/views/MapView.vue`

原 WebGIS 底图页面，结构复杂但核心可用：

- MapLibre raster 底图：`/tiles/raster/{z}/{x}/{y}.png`
- MapLibre vector 图层：coastline、rivers、ecoregions
- Deck.gl overlay：PathLayer、TripsLayer、ScatterplotLayer
- Pinia 控制互斥选择：node / route / ecozone
- Dev Tools 控制 L0-L3 图层可见性

注意：

- `MapLibre addLayer(spec, beforeId)` 是插入到 `beforeId` 之前，不要把矢量层误插到 raster 下面。
- 后端启动可能很慢，首次会解压/加载 GIS 数据。
- 地图页面依赖 FastAPI 和本地 GIS 数据，单独开 Vite 可能无法完整显示底图。

## 当前风险和坑

- 项目中文文件在部分 PowerShell 输出中会乱码，但源码和文档本身仍是可读的；编辑时保持 UTF-8。
- `.env` 包含数据库配置且被 gitignore，不能打印或提交真实凭据。
- `dist/`、`node_modules/`、大数据文件和临时截图不应作为前端改动一起提交。
- `/brand` 是超大单文件，继续追加 CSS 会越来越难维护；但短期为了交付可以继续做局部修补。
- 真实地图渲染依赖后端 `/tiles/*`，生产部署时必须确保 Express 或反向代理能访问 FastAPI。
