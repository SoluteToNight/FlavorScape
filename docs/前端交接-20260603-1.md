## 📁 一、 前端项目源码文件架构

整个前端基于 Vue 3 + Vite 构建，采用扁平化的页面管理结构，核心运行代码及资源完全收拢在 `src/` 和 `public/` 目录下：

### 🧱 核心数据流与联动逻辑

- **状态同步中心**：所有平级核心视图（`BrandStoryView`、`MarketingView` 等）统一遵循并读写 `stores/app.js`（Pinia）的全局状态，实现“顶栏切网页，上下文产品与节点数据不丢失”的顺滑体验。
- **前后端调用关系**：开发环境（`5173`）通过 Vite 代理，将 `/api/*` 和 `/tiles/*` 骨干网络请求转发至 FastAPI 后端（`8001`）。生产环境下由 `server.js` 托管 `dist/` 目录。

## 📌 二、 当前最新重构进度

1. **营销海报模块独立解耦**：成功在 `src/views/` 下创建平级路由视图 **`MarketingView.vue`**，彻底解决了之前多视图在单文件内混乱堆叠导致的 CSS 覆盖与模糊性。
2. **静态地图组件落地**：全新编写了 `src/components/StaticGeoMap.vue`，利用 `d3-geo` 离线投影技术动态读取 `assets/china.json`。地图可根据食材自动高亮省份并画出航班图式的平滑溯源连线，**100% 根除了 `html2canvas` 导出海报时 WebGL 变黑/留白的行业经典 Bug**。
3. **多骨相高级模板机制**：海报内集成了 4 套美学模板（自然画册/东方文人/风味蓝图/沉浸纪实），切换时不只是变色，DOM 骨架与排版也会彻底改变。
4. **图片焦点滑块**：新增了无损图片视口调节功能（CSS `object-position` 联动原生 range 滑块），允许用户动态替换大图并像素级调整画面主体的显示焦点。

## 🚀 三、 下一步核心开发任务规划 (Next Tasks)

后续对话需要立即承接重构势头，对剩下的模块执行清洗、更名并独立构建开发：

### 🛠️ 任务一：新加平级路由与顶栏功能性引导

- **更名策略**：原本的“科学档案/风物档案”一词由于面向消费者的目的不够具体，现对其进行全新产品化定义——建议更名为更具市场说服力与吸眼球的 **“风土风味看板”**、**“数据实证舱”** 或 **“科学溯源白皮书 (Provenance Dashboard)”**。
- **具体操作**：
  1. 在 `src/router/index.js` 中，注册一个平级路由，例如 `{ path: '/archive', name: 'archive', component: () => import('../views/ArchiveView.vue') }`。
  2. 在 `src/components/Navbar.vue` 顶栏中新加对应的选项，引导用户一键进入科学实证数据中心。

### 🛠️ 任务二：全新设计与开发“科学数据页” (`ArchiveView.vue`)

- **具体操作**：在 `src/views/` 目录下新建全新的平级独立视图组件 **`ArchiveView.vue`**。
- **核心呈现要素**：该页面彻底放弃任何散文式画册排版，面向**硬核消费者与采购质检员**，全面引入 **`ECharts`** 进行科学实证可视化，并显著增强地图与数据的有效叙事联动：
  1. **气候风土因子数据**：渲染出产区“月均降水量 vs 均温”的双轴折线图、“生长期有效积温”柱状图。
  2. **品质与食品安全硬核证据**：渲染出关键成分（如花椒挥发油、大米直链淀粉指标）超越国标的对比雷达图（可调用 `FlavorRadar.vue`），并设置专属区域展示官方 SGS / 谱尼测试的农残 0 检出合格报告块。
  3. **地图联动**：让地图在其中退化为清晰的“数据采样点”或“气象站”分布图，做到数据极度精准、一眼建立硬核信任。
