# 科学数据页开发进度更新 - 2026-06-03

## 本次完成内容

本次围绕“科学溯源白皮书 / Provenance Dashboard”完成了第一版功能框架搭建，目标是把原先偏画册式的档案概念升级为面向硬核消费者、采购质检员和品牌方的科学实证数据页。

### 1. 新增平级路由

- 在 `src/router/index.js` 中新增 `/archive` 平级路由。
- 路由组件指向 `src/views/ArchiveView.vue`。

### 2. 新增顶栏入口

- 在 `src/components/Navbar.vue` 中新增导航项“实证白皮书”。
- `archive` 页面已加入产品化页面透明顶栏状态判断。

### 3. 新建 ArchiveView.vue

新增 `src/views/ArchiveView.vue`，目前使用本地 mock 数据搭建页面框架，后续可替换为后端接口数据。

已包含模块：

- 产区样本切换：汉源花椒、五常大米。
- 气候风土因子：月均降水量 vs 均温双轴折线图。
- 生长期数据：有效积温柱状图。
- 品质证据：关键成分对标国标雷达图。
- 风味摘要：复用 `FlavorRadar.vue` 展示风味指纹。
- 食品安全证据：SGS / 谱尼测试报告块，展示农残 0 检出。
- 产地地图：改为静态地形底图，不再使用抽象矢量地图。

### 4. 地图改造

新增页地图框已从 `StaticGeoMap` 替换为轻量 MapLibre 静态地图：

- 底图复用原 WebGIS 的地形瓦片源：`/tiles/raster/{z}/{x}/{y}.png`。
- 省级边界数据来自 `src/assets/china.json`。
- 页面根据当前样本的 `province` 字段过滤并高亮对应省份。
- 标注产地对应县级/县级市行政中心位置：
  - 汉源花椒：四川省雅安市汉源县行政中心近似坐标。
  - 五常大米：黑龙江省哈尔滨市五常市行政中心近似坐标。

## 后续建议

- 将 `ArchiveView.vue` 中的 mock 数据拆到独立数据文件或后端接口。
- 进一步补充真实检测报告字段，如报告日期、检测机构编号、检测项目明细、PDF/图片链接。
- 若需要县级行政区边界高亮，需要补充更细粒度的市县级 GeoJSON；当前 `src/assets/china.json` 只有省级边界。
- 后续可把静态地图封装为独立组件，例如 `ProvenanceTerrainMap.vue`，供更多科学数据页或产地详情页复用。



补充：这个页面开发的初衷和理念：
这个页面（新更名的独立页面）应当面向两类最挑剔的硬核人群：  

1. **线上的精明消费者（母婴社群、成分党、高净值中产）**：
   - **他们的痛点**：现在市场上到处都在讲“地道、纯天然、有机”，全凭商家一张嘴，消费者早就免疫了。
   - **我们的解法**：他们想买五常大米，点进这个页面，不看虚的情怀，直接看这个大米在 8 月灌浆期时的**真实昼夜温差图表**和 **SGS 官方农残 0 检出报告**。用铁一样的数据，支撑他们多花一倍钱购买的决策。  
2. **B端的精品电商采购员 / 高级餐厅主厨**：
   - **他们的痛点**：选品需要向老板和合规部门汇报，需要客观的准入证据。
   - **我们的解法**：直接把这个页面当作一份“电子质检报告”，里面有该批次食材的微观县级遥感产地验证和理化风味指标（如花椒挥发油测定值）。
