# 食材历史传播数据结构 v1

本文档用于整理 `data/ingredient/*.json`。目标是让不同食材的文献线索、地理节点、传播路径和前端镜头叙事都能按同一套结构录入。

## 设计原则

- 坐标统一使用 WGS84，经纬度顺序为 `[longitude, latitude]`。
- 每个时间点必须能回到文献或研究出处；不确定推断也要写明依据和置信度。
- `timeline` 按时间排序即可完成基础可视化；复杂分叉路径可额外填写 `segments`。
- 年代不确定时，用 `year` 填代表年份，另用 `year_range` 和 `period_label` 保存真实表述。

## 顶层字段

```json
{
  "ingredient_id": "pepper",
  "name": "辣椒",
  "name_en": "Chili Pepper",
  "species": "Capsicum annuum",
  "origin": "美洲",
  "origin_coordinates": [-99.13, 19.43],
  "summary": "用于页面摘要的一段话。",
  "color": "#E5394E",
  "tags": ["茄科", "调味料", "大航海时代"],
  "timeline": [],
  "segments": [],
  "references": []
}
```

## timeline 事件

`timeline` 是当前前端默认读取的核心结构。每个事件表示食材在某地被记录、引入、种植、商品化或进入地方饮食的证据点。

```json
{
  "event_id": "pepper-1591-zhejiang",
  "year": 1591,
  "year_range": [1591, 1591],
  "period_label": "明代后期",
  "dynasty": "明代后期",
  "location": "浙江",
  "coordinates": [120.15, 30.28],
  "historical_name": ["番椒"],
  "event_type": "首次记载",
  "route": "华东沿海传入",
  "transport_mode": "sea",
  "confidence": "high",
  "notes": "中国目前公认的辣椒最早记载。",
  "source_literature": "《遵生八笺》",
  "source_refs": ["ref-gaolian-zunshengbajian"],
  "source_excerpt": "可摘录短句，注意版权和引用长度。",
  "visual": {
    "camera_zoom": 4.8,
    "importance": 1
  }
}
```

推荐枚举：

- `event_type`: `起源`, `跨洋传播`, `首次记载`, `早期传播`, `区域扩散`, `核心区形成`, `饮食演变`, `命名变化`, `晚期补证`
- `transport_mode`: `sea`, `land`, `river`, `caravan`, `colonial_trade`, `local_diffusion`, `unknown`
- `confidence`: `high`, `medium`, `low`

## segments 路径段

不填 `segments` 时，前端按 `origin_coordinates -> timeline[0] -> timeline[1]...` 自动生成路径。若后续需要表达分叉、回流、海陆复合路线或多文献路线，建议显式填写。

```json
{
  "segment_id": "pepper-atlantic-1493",
  "from_event_id": "origin",
  "to_event_id": "pepper-1493-spain",
  "route_name": "哥伦布第二次航行",
  "route_type": "sea",
  "geometry": {
    "type": "LineString",
    "coordinates": [
      [-99.13, 19.43],
      [-65.0, 17.0],
      [-3.70, 40.42]
    ]
  },
  "start_year": 1492,
  "end_year": 1493,
  "confidence": "medium",
  "evidence_note": "依据航海日志和后续植物传播研究推断。",
  "source_refs": ["ref-columbus-journal"]
}
```

## references 文献

```json
{
  "ref_id": "ref-gaolian-zunshengbajian",
  "title": "遵生八笺",
  "author": "高濂",
  "year": 1591,
  "type": "古籍",
  "publisher": "",
  "url": "",
  "note": "记录番椒。"
}
```

## 录入检查清单

- 每个事件都有 `year`, `location`, `coordinates`, `event_type`, `source_literature`。
- 推断路线必须写 `confidence` 和 `evidence_note` 或 `notes`。
- 地名使用现代可定位名称，历史名称放在 `historical_name`。
- 同一年多个事件时使用唯一 `event_id`，不要只靠年份识别。
- 文献出处优先放在 `references`，事件内用 `source_refs` 关联。
